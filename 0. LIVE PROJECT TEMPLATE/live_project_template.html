💼 Real Model Example: JobPost

jobpost/models.py
Python 
class JobPost(models.Model):
    """Main model for job postings with all relationship types"""
    
    JOB_TYPE_CHOICES = [
        ('Full Time', 'Full Time'),
        ('Part Time', 'Part Time'),
    ]
    
    # Basic Fields
    title = models.CharField(max_length=200)
    description = models.CharField(max_length=200)
    salary = models.IntegerField()
    slug = models.SlugField(max_length=40, unique=True, null=True)
    type = models.CharField(max_length=200, choices=JOB_TYPE_CHOICES, default='Full Time')
    
    # Date Fields
    date = models.DateTimeField(auto_now_add=True)  # Set on creation
    expiry = models.DateField(null=True)           # Manual date
    
    # Relationships
    location = models.OneToOneField(Location, on_delete=models.CASCADE, null=True)
    author = models.ForeignKey(Author, on_delete=models.CASCADE, null=True)
    skills = models.ManyToManyField(Skills, related_name='jobpost')

    def __str__(self):
        return self.title

🔗 Understanding Relationships

1↔1
OneToOneField
JobPost↔Location

One job = One location. Each location belongs to one job.

location = models.OneToOneField(Location, on_delete=models.CASCADE)
∞→1
ForeignKey
JobPost (Many)→Author (One)

Many jobs belong to one author. One author has many jobs.

author = models.ForeignKey(Author, on_delete=models.CASCADE)
∞↔∞
ManyToManyField
JobPost↔Skills

Many jobs can have many skills. Skills can be in many jobs.

skills = models.ManyToManyField(Skills, related_name='jobpost')

🔍 Django ORM Queries Cheatsheet

Django Shell Examples
Python 
# ==========================================
# CRUD Operations
# ==========================================

# CREATE
job = JobPost.objects.create(title='Python Dev', salary=80000)

# READ
all_jobs = JobPost.objects.all()              # Get all
job = JobPost.objects.get(id=1)               # Get single by ID
job = JobPost.objects.get(slug='python-dev')  # Get by field

# UPDATE
job.salary = 85000
job.save()

# DELETE
job.delete()

# ==========================================
# Filtering & Lookups
# ==========================================

JobPost.objects.filter(type='Full Time')       # Exact match
JobPost.objects.filter(salary__gte=70000)      # >= 70000
JobPost.objects.filter(salary__lte=100000)     # <= 100000
JobPost.objects.filter(title__icontains='python')  # Contains (case-insensitive)
JobPost.objects.filter(date__year=2024)        # By year
JobPost.objects.exclude(type='Part Time')      # Not equal

# ==========================================
# Relationship Queries
# ==========================================

# Forward (job → author)
job.author.company                             # Access related field

# Reverse (author → jobs)
author.jobpost.all()                           # All jobs by author

# ManyToMany
job.skills.all()                               # All skills for job
job.skills.add(skill1, skill2)                 # Add skills

# Filter by related field
JobPost.objects.filter(author__company='ABC Corp')
JobPost.objects.filter(skills__name='Python')

# ==========================================
# Ordering & Aggregation
# ==========================================

JobPost.objects.order_by('-salary')            # Descending
JobPost.objects.all()[:5]                      # First 5

from django.db.models import Count, Avg
JobPost.objects.aggregate(Avg('salary'))       # Average

from django.http import JsonResponse
from .models import JobPost, Skills

def jobpost_list(request):
    """Return all job posts as JSON"""
    jobs = JobPost.objects.all()
    data = [
        {
            'title': job.title,
            'salary': job.salary,
            'author': str(job.author) if job.author else None,
            'skills': [s.name for s in job.skills.all()],
        }
        for job in jobs
    ]
    return JsonResponse({'jobs': data})

def jobpost_detail(request, job_id):
    """Return single job by ID"""
    try:
        job = JobPost.objects.get(id=job_id)
        return JsonResponse({'title': job.title, 'salary': job.salary})
    except JobPost.DoesNotExist:
        return JsonResponse({'error': 'Not found'}, status=404)

🔧 Common AWS Issues & Solutions

❌ Connection Timeout to EC2
Problem: Cannot SSH or connect to EC2 instance
Solutions:
  • Check Security Group allows inbound port 22 (SSH) or 80/443 (HTTP/HTTPS)
  • Verify Network ACL rules in VPC subnet
  • Ensure instance has public IP if connecting from internet
  • Check SSH key permissions: chmod 400 key.pem
  • Verify instance is in running state
⚠️ RDS Connection Refused
Problem: Django cannot connect to RDS database
Solutions:
  • Verify RDS Security Group allows inbound port 5432 (PostgreSQL)
  • Check EC2 instance is in same VPC or has proper peering
  • Use RDS endpoint (not IP address) in Django settings
  • Verify database credentials are correct
  • Check RDS is in available state
⚠️ S3 Access Denied (403)
Problem: Django cannot upload/download files from S3
Solutions:
  • Verify IAM role attached to EC2 has S3 permissions
  • Check bucket policy allows necessary actions
  • Ensure AWS credentials are properly configured
  • Verify bucket name and region in settings
  • Check for Block Public Access settings if needed
💡 High AWS Costs
Problem: Unexpected high monthly bills
Solutions:
  • Use AWS Cost Explorer to identify expensive services
  • Enable AWS Budgets with alerts
  • Stop unused EC2 instances (dev/test environments)
  • Delete old EBS snapshots and AMIs
  • Use S3 lifecycle policies to move old data to Glacier
  • Consider Reserved Instances for production workloads
❌ CloudFront Not Updating
Problem: Static files not updating after deployment
Solutions:
  • Create CloudFront invalidation: aws cloudfront create-invalidation --distribution-id XXX --paths "/*"
  • Wait 5-10 minutes for cache invalidation to propagate
  • Set appropriate Cache-Control headers in S3
  • Use versioned filenames for static assets
  • Check CloudFront distribution status is deployed

import boto3
from PIL import Image
import io

s3 = boto3.client('s3')

def lambda_handler(event, context):
    """Generate thumbnail when image uploaded to S3"""
    bucket = event['Records'][0]['s3']['bucket']['name']
    key = event['Records'][0]['s3']['object']['key']
    
    # Download image
    response = s3.get_object(Bucket=bucket, Key=key)
    image = Image.open(io.BytesIO(response['Body'].read()))
    
    # Create thumbnail
    image.thumbnail((200, 200))
    buffer = io.BytesIO()
    image.save(buffer, 'JPEG')
    buffer.seek(0)
    
    # Upload thumbnail
    thumbnail_key = f"thumbnails/{key}"
    s3.put_object(
        Bucket=bucket,
        Key=thumbnail_key,
        Body=buffer,
        ContentType='image/jpeg'
    )
    
    return {'statusCode': 200, 'body': 'Thumbnail created'}

🔧 Common AWS Issues & Solutions

❌ Connection Timeout to EC2
Problem: Cannot SSH or connect to EC2 instance
Solutions:
  • Check Security Group allows inbound port 22 (SSH) or 80/443 (HTTP/HTTPS)
  • Verify Network ACL rules in VPC subnet
  • Ensure instance has public IP if connecting from internet
  • Check SSH key permissions: chmod 400 key.pem
  • Verify instance is in running state
⚠️ RDS Connection Refused
Problem: Django cannot connect to RDS database
Solutions:
  • Verify RDS Security Group allows inbound port 5432 (PostgreSQL)
  • Check EC2 instance is in same VPC or has proper peering
  • Use RDS endpoint (not IP address) in Django settings
  • Verify database credentials are correct
  • Check RDS is in available state
⚠️ S3 Access Denied (403)
Problem: Django cannot upload/download files from S3
Solutions:
  • Verify IAM role attached to EC2 has S3 permissions
  • Check bucket policy allows necessary actions
  • Ensure AWS credentials are properly configured
  • Verify bucket name and region in settings
  • Check for Block Public Access settings if needed
💡 High AWS Costs
Problem: Unexpected high monthly bills
Solutions:
  • Use AWS Cost Explorer to identify expensive services
  • Enable AWS Budgets with alerts
  • Stop unused EC2 instances (dev/test environments)
  • Delete old EBS snapshots and AMIs
  • Use S3 lifecycle policies to move old data to Glacier
  • Consider Reserved Instances for production workloads
❌ CloudFront Not Updating
Problem: Static files not updating after deployment
Solutions:
  • Create CloudFront invalidation: aws cloudfront create-invalidation --distribution-id XXX --paths "/*"
  • Wait 5-10 minutes for cache invalidation to propagate
  • Set appropriate Cache-Control headers in S3
  • Use versioned filenames for static assets
  • Check CloudFront distribution status is deployed
