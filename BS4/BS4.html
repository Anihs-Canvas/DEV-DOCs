Kinds of Objects

Beautiful Soup transforms a complex HTML document into a tree of Python objects. There are four main kinds of objects you'll encounter: Tag, NavigableString, BeautifulSoup, and Comment. Understanding these objects is fundamental to effective web scraping.

🔍 Object Types Overview

Tag - HTML/XML tag elements
NavigableString - Text content within tags
BeautifulSoup - The document as a whole
Comments - HTML comments
Doctype - Document type declarations
Name - Tag name attribute
💡 Key Concept: Objects returned as lists are treated (CRUD) like lists. Objects returned as dictionaries are treated like dictionaries. Beautiful Soup uses native Python data structures.

Tag

Docs: Beautiful Soup Tag Documentation

A Tag object corresponds to an XML or HTML tag in the original document.

Full description

Tags are the main type of object you'll work with in Beautiful Soup. Every HTML element is represented as a Tag object with properties like name, attributes, and children. Tags can be searched, modified, and navigated.

Syntax

tag = soup.find('tag_name')

Example HTML Document

Alice in Wonderland Example
html_doc = """
<html><head><title>The Dormouse's story</title></head>
<body>
<p class="title"><b>The Dormouse's story</b></p>

<p class="story">Once upon a time there were three little sisters; and their names were
<a href="http://example.com/elsie" class="sister" id="link1">Elsie</a>,
<a href="http://example.com/lacie" class="sister" id="link2">Lacie</a> and
<a href="http://example.com/tillie" class="sister" id="link3">Tillie</a>;
and they lived at the bottom of a well.</p>

<p class="story">...</p>
"""

from bs4 import BeautifulSoup
soup = BeautifulSoup(html_doc, 'html.parser')

Examples

Example 1: Finding a tag
from bs4 import BeautifulSoup

soup = BeautifulSoup(html_doc, 'html.parser')
tag = soup.find('title')
print(tag)
print(type(tag))
<title>The Dormouse's story</title> <class 'bs4.element.Tag'>
Example 2: Accessing tag by name
# Access tag directly using attribute syntax
title_tag = soup.title
print(title_tag)

# Access nested tags
b_tag = soup.b
print(b_tag)
<title>The Dormouse's story</title> <b>The Dormouse's story</b>
Example 3: Finding multiple tags
# Find all anchor tags
all_links = soup.find_all('a')
print(f"Found {len(all_links)} links:")
for link in all_links:
    print(link)
Found 3 links: <a class="sister" href="http://example.com/elsie" id="link1">Elsie</a> <a class="sister" href="http://example.com/lacie" id="link2">Lacie</a> <a class="sister" href="http://example.com/tillie" id="link3">Tillie</a>

NavigableString

Docs: NavigableString Documentation

A string corresponds to a bit of text within a tag.

Full description

NavigableString is used to represent text content within tags. It behaves like a Python string but has extra features for navigating the document tree. You can access it via the .string or .text attributes.

Syntax

navigable_string = tag.string
navigable_string = tag.text
navigable_string = NavigableString("Some text")

Examples

Example 1: Getting string from a tag
title_tag = soup.title
text = title_tag.string
print(text)
print(type(text))
The Dormouse's story <class 'bs4.element.NavigableString'>
Example 2: Using .text attribute
b_tag = soup.b
text = b_tag.text
print(text)
The Dormouse's story
Example 3: Creating a NavigableString
from bs4 import NavigableString

new_string = NavigableString("This is new text")
print(new_string)
print(type(new_string))
This is new text <class 'bs4.element.NavigableString'>

BeautifulSoup

Docs: BeautifulSoup Object Documentation

The BeautifulSoup object represents the parsed document as a whole.

Full description

The BeautifulSoup object is the main container that holds your entire parsed document. It supports searching and navigating the document tree. Think of it as a special kind of Tag that represents the document itself.

Syntax

soup = BeautifulSoup(markup, features)

Parameters

  • markup: The HTML or XML document to parse (string or file-like object)
  • features: The parser to use ('html.parser', 'lxml', 'html5lib', etc.)

Examples

Example 1: Creating a BeautifulSoup object
from bs4 import BeautifulSoup

html = "<html><body><p>Hello World</p></body></html>"
soup = BeautifulSoup(html, 'html.parser')
print(type(soup))
print(soup.name)
<class 'bs4.BeautifulSoup'> [document]
Example 2: Parsing from a file
with open('index.html', 'r', encoding='utf-8') as file:
    soup = BeautifulSoup(file, 'html.parser')
    print(soup.title.string)
The Dormouse's story
Example 3: Different parsers
# HTML parser (built-in)
soup1 = BeautifulSoup(html_doc, 'html.parser')

# lxml parser (fast)
soup2 = BeautifulSoup(html_doc, 'lxml')

# html5lib parser (lenient)
soup3 = BeautifulSoup(html_doc, 'html5lib')

print(f"html.parser: {soup1.name}")
print(f"lxml: {soup2.name}")
print(f"html5lib: {soup3.name}")
html.parser: [document] lxml: [document] html5lib: [document]

Comments

Docs: Comments Documentation

The Comment class represents HTML comments in the document.

Full description

Comments are special NavigableString objects that represent HTML comments (<!-- comment -->). You can create, find, and manipulate comments just like other strings.

Syntax

from bs4 import Comment
comment = Comment("text")
comment = soup.find(string=lambda text: isinstance(text, Comment))

Examples

Example 1: Finding comments
from bs4 import BeautifulSoup, Comment

html = "<html><!-- This is a comment --><body><p>Text</p></body></html>"
soup = BeautifulSoup(html, 'html.parser')

comment = soup.find(string=lambda text: isinstance(text, Comment))
print(comment)
print(type(comment))
This is a comment <class 'bs4.element.Comment'>
Example 2: Creating a comment
from bs4 import Comment

new_comment = Comment("This is a new comment")
print(new_comment)
print(type(new_comment))
This is a new comment <class 'bs4.element.Comment'>
Example 3: Finding all comments
html = """
<div>
    <!-- Comment 1 -->
    <p>Text</p>
    <!-- Comment 2 -->
</div>
"""
soup = BeautifulSoup(html, 'html.parser')
comments = soup.find_all(string=lambda text: isinstance(text, Comment))
for comment in comments:
    print(f"Found: {comment.strip()}")
Found: Comment 1 Found: Comment 2

The Doctype class represents the document type declaration.

The Doctype object represents the <!DOCTYPE> declaration at the beginning of an HTML document. It's a special type of NavigableString.

from bs4 import Doctype
doctype = soup.find_all(string=lambda text: isinstance(text, Doctype))

from bs4 import BeautifulSoup, Doctype

html = "<!DOCTYPE html><html><body><p>Hello</p></body></html>"
soup = BeautifulSoup(html, 'html.parser')

doctype = soup.find(string=lambda text: isinstance(text, Doctype))
print(doctype)
print(type(doctype))
html <class 'bs4.element.Doctype'>

Every tag has a name, accessible via the .name attribute.

The .name attribute gives you the tag's name as a string. You can also modify a tag's name by assigning a new value to this attribute.

Syntax

tag_name = tag.name
tag.name = "new_tag_name"

Examples

Example 1: Getting tag name
title_tag = soup.title
print(f"Tag name: {title_tag.name}")

b_tag = soup.b

Tag name: title Tag name: b
Example 2: Changing tag name
tag = soup.b
print(f"Before: {tag}")

tag.name = "strong"
print(f"After: {tag}")
Before: <b>The Dormouse's story</b> After: <strong>The Dormouse's story</strong>

Tag Operations

Tags have attributes and strings that you can access and modify. Understanding how to work with attributes (returned as dictionaries) and multi-valued attributes (returned as lists) is essential for effective web scraping.

🏷️ Tag Operations Overview

Attributes - Dict-like access to tag attributes
Multi-valued - List-like class, rel attributes
NavigableString - Working with text content

Attributes

Docs: Attributes Documentation

Tag attributes are stored in a dictionary and can be accessed like a dictionary.

Full description

Every tag can have attributes (like class, id, href). These are returned as a dictionary via .attrs. You can access, modify, and delete attributes using standard dictionary operations.

Syntax

attributes = tag.attrs
attribute_value = tag['attribute_name']
tag['attribute_name'] = 'new_value'
del tag['attribute_name']

Examples

Example 1: Getting all attributes
link = soup.find('a')
print(f"All attributes: {link.attrs}")
print(f"Type: {type(link.attrs)}")
All attributes: {'href': 'http://example.com/elsie', 'class': ['sister'], 'id': 'link1'} Type: <class 'dict'>
Example 2: Accessing specific attribute
link = soup.find('a')
print(f"href: {link['href']}")
print(f"id: {link['id']}")
print(f"class: {link['class']}")
href: http://example.com/elsie id: link1 class: ['sister']
Example 3: Modifying attributes
link = soup.find('a')
print(f"Before: {link}")

link['href'] = 'http://newurl.com'
link['target'] = '_blank'
print(f"After: {link}")
Before: <a class="sister" href="http://example.com/elsie" id="link1">Elsie</a> After: <a class="sister" href="http://newurl.com" id="link1" target="_blank">Elsie</a>
Example 4: Deleting attributes
link = soup.find('a')
print(f"Before: {link}")

del link['id']
print(f"After: {link}")
Before: <a class="sister" href="http://example.com/elsie" id="link1">Elsie</a> After: <a class="sister" href="http://example.com/elsie">Elsie</a>
Example 5: Using .get() method
link = soup.find('a')
# Safe way to get attributes (returns None if not found)
href = link.get('href')
title = link.get('title', 'Default Title')
print(f"href: {href}")
print(f"title: {title}")
href: http://example.com/elsie title: Default Title

Multi-valued Attributes

Docs: Multi-valued Attributes Documentation

Some attributes like 'class' can have multiple values and are returned as lists.

Full description

HTML attributes like class and rel can contain multiple values. Beautiful Soup returns these as lists. You can treat them like regular Python lists for CRUD operations.

Syntax

multi_valued_attribute = tag['attribute_name']
multi_valued_attribute = tag.get('attribute_name')
tag['attribute_name'] = ['value1', 'value2']

Examples

Example 1: Accessing class attribute
link = soup.find('a')
classes = link['class']
print(f"Classes: {classes}")
print(f"Type: {type(classes)}")
Classes: ['sister'] Type: <class 'list'>
Example 2: Multiple classes
html = '<div class="container main-content featured">Text</div>'
soup = BeautifulSoup(html, 'html.parser')
div = soup.div
print(f"Classes: {div['class']}")
print(f"First class: {div['class'][0]}")
print(f"Number of classes: {len(div['class'])}")
Classes: ['container', 'main-content', 'featured'] First class: container Number of classes: 3
Example 3: Modifying class list
html = '<div class="old-class">Text</div>'
soup = BeautifulSoup(html, 'html.parser')
div = soup.div

# Add a class
div['class'].append('new-class')
print(f"After append: {div}")

# Replace all classes
div['class'] = ['class1', 'class2', 'class3']
print(f"After replace: {div}")
After append: <div class="old-class new-class">Text</div> After replace: <div class="class1 class2 class3">Text</div>
Example 4: Working with rel attribute
html = '<link rel="stylesheet prefetch" href="style.css">'
soup = BeautifulSoup(html, 'html.parser')
link = soup.link
print(f"rel attribute: {link['rel']}")
print(f"Type: {type(link['rel'])}")
rel attribute: ['stylesheet', 'prefetch'] Type: <class 'list'>

Navigating the Tree

Beautiful Soup provides many attributes and methods for navigating the parse tree. You can move down to children, up to parents, sideways to siblings, and back and forth in parse order.

🧭 Navigation Overview

Going Down - .contents, .children, .descendants
Going Up - .parent, .parents
Going Sideways - .next_sibling, .previous_sibling
Back & Forth - .next_element, .previous_element

Going Down

Navigate down the tree to access child elements.

Full description

Moving down the tree means accessing a tag's children and descendants. Use .contents for a list, .children for an iterator, and .descendants for all nested tags recursively.

.contents

tag.contents

Return Value: List of Tag and NavigableString objects representing direct children.

Example: Using .contents
head = soup.head
print(f"Contents: {head.contents}")
print(f"Type: {type(head.contents)}")
print(f"First child: {head.contents[0]}")
Contents: [<title>The Dormouse's story</title>] Type: <class 'list'> First child: <title>The Dormouse's story</title>

.children

tag.children

Return Value: Iterator (generator) yielding each direct child.

Example: Using .children
body = soup.body
print(f"Type: {type(body.children)}")
print("Children:")
for i, child in enumerate(body.children, 1):
    if child.name:
        print(f"  {i}. {child.name}: {str(child)[:50]}...")
Type: <class 'list_iterator'> Children: 1. p: <p class="title"><b>The Dormouse's story</b></p>... 2. p: <p class="story">Once upon a time there were th...

.descendants

tag.descendants

Return Value: Generator yielding all descendants recursively.

Example: Using .descendants
head = soup.head
print("Descendants:")
for i, descendant in enumerate(head.descendants, 1):
    if hasattr(descendant, 'name'):
        print(f"  {i}. {descendant.name if descendant.name else 'text'}: {str(descendant)[:40]}")
Descendants: 1. title: <title>The Dormouse's story</title> 2. text: The Dormouse's story

.string

tag.string

Return Value: NavigableString if tag has only one child, None if multiple children.

Example: Using .string
title = soup.title
print(f"Title string: {title.string}")

# Tag with multiple children returns None
body = soup.body
print(f"Body string: {body.string}")
Title string: The Dormouse's story Body string: None

.strings

tag.strings

Return Value: Generator yielding all NavigableString objects.

Example: Using .strings
body = soup.body
print("All strings:")
for i, string in enumerate(body.strings, 1):
    print(f"  {i}. {repr(string[:30])}")
All strings: 1. "The Dormouse's story" 2. '\n' 3. '\n' 4. 'Once upon a time there were th'

.stripped_strings

tag.stripped_strings

Return Value: Generator yielding NavigableString objects with whitespace removed.

Example: Using .stripped_strings
body = soup.body
print("Stripped strings:")
for i, string in enumerate(body.stripped_strings, 1):
    if i <= 5:
        print(f"  {i}. {string[:40]}")
Stripped strings: 1. The Dormouse's story 2. Once upon a time there were three litt 3. Elsie 4. , 5. Lacie

Going Up

Navigate up the tree to access parent elements.

Full description

Moving up the tree means accessing a tag's parent and all ancestors. Use .parent for immediate parent and .parents for all ancestors.

.parent

tag.parent

Return Value: Tag object representing the immediate parent.

Example: Using .parent
title = soup.title
print(f"Title tag: {title}")
print(f"Parent: {title.parent.name}")
print(f"Grandparent: {title.parent.parent.name}")
Title tag: <title>The Dormouse's story</title> Parent: head Grandparent: html

.parents

tag.parents

Return Value: Generator yielding all parent Tag objects up to the root.

Example: Using .parents
link = soup.find('a')
print(f"Link: {link['id']}")
print("All parents:")
for i, parent in enumerate(link.parents, 1):
    if parent.name:
        print(f"  {i}. {parent.name}")
Link: link1 All parents: 1. p 2. body 3. html 4. [document]

Going Sideways

Navigate to sibling elements at the same level.

Full description

Siblings are tags at the same level of the tree. Use .next_sibling and .previous_sibling to move between adjacent siblings.

.next_sibling

tag.next_sibling

Return Value: Next sibling Tag or NavigableString, or None if none exists.

Example: Using .next_sibling
first_link = soup.find('a')
print(f"First link: {first_link.string}")
print(f"Next sibling: {repr(first_link.next_sibling)}")
First link: Elsie Next sibling: ','

.previous_sibling

tag.previous_sibling

Return Value: Previous sibling Tag or NavigableString, or None.

Example: Using .previous_sibling
second_link = soup.find('a', id='link2')
print(f"Second link: {second_link.string}")
prev = second_link.previous_sibling
print(f"Previous sibling: {repr(prev)}")
Second link: Lacie Previous sibling: ',\n'

.next_siblings

tag.next_siblings

Return Value: Iterator over all next siblings.

Example: Using .next_siblings
first_link = soup.find('a')
print(f"First link: {first_link.string}")
print("Next siblings that are tags:")
for sibling in first_link.next_siblings:
    if hasattr(sibling, 'name') and sibling.name:
        print(f"  - {sibling.name}: {sibling.string}")
First link: Elsie Next siblings that are tags: - a: Lacie - a: Tillie

.previous_siblings

tag.previous_siblings

Return Value: Iterator over all previous siblings.

Example: Using .previous_siblings
last_link = soup.find('a', id='link3')
print(f"Last link: {last_link.string}")
print("Previous siblings that are tags:")
for sibling in last_link.previous_siblings:
    if hasattr(sibling, 'name') and sibling.name:
        print(f"  - {sibling.name}: {sibling.string}")
Last link: Tillie Previous siblings that are tags: - a: Lacie - a: Elsie

Going Back and Forth

Navigate in parse order through the document.

Full description

Parse order navigation moves through elements in the order they appear in the document, regardless of tree structure. Use .next_element and .previous_element.

.next_element

element.next_element

Return Value: Next element in parse order.

Example: Using .next_element
title = soup.title
print(f"Current: {title}")
print(f"Next element: {repr(title.next_element)}")
Current: <title>The Dormouse's story</title> Next element: "The Dormouse's story"

.previous_element

element.previous_element

Return Value: Previous element in parse order.

Example: Using .previous_element
link = soup.find('a')
print(f"Current: {link.string}")
print(f"Previous element: {repr(link.previous_element)}")
Current: Elsie Previous element: 'Once upon a time there were three little sisters; and their names were\n'

.next_elements

element.next_elements

Return Value: Generator yielding all next elements in parse order.

Example: Using .next_elements
title = soup.title
print("Next 5 elements:")
for i, elem in enumerate(title.next_elements, 1):
    if i <= 5:
        if hasattr(elem, 'name') and elem.name:
            print(f"  {i}. Tag: {elem.name}")
        else:
            print(f"  {i}. Text: {repr(str(elem)[:30])}")
Next 5 elements: 1. Text: "The Dormouse's story" 2. Tag: body 3. Text: '\n' 4. Tag: p 5. Tag: b

.previous_elements

element.previous_elements

Return Value: Generator yielding all previous elements in parse order.

Example: Using .previous_elements
link = soup.find('a')
print("Previous 5 elements:")
for i, elem in enumerate(link.previous_elements, 1):
    if i <= 5:
        if hasattr(elem, 'name') and elem.name:
            print(f"  {i}. Tag: {elem.name}")
        else:
            print(f"  {i}. Text: {repr(str(elem)[:20])}")
Previous 5 elements: 1. Text: 'Once upon a time the' 2. Tag: p 3. Text: '\n' 4. Tag: b 5. Text: "The Dormouse's story"

Searching the Tree

Beautiful Soup provides powerful search methods to find tags and text. You can use strings, regular expressions, lists, boolean values, and custom functions as filters.

🔍 Search Methods Overview

find_all() - Find all matching elements
find() - Find first match
find_parents() - Search upward
find_next_siblings() - Search forward
find_previous_siblings() - Search backward
select() - CSS selectors

Kinds of Filters

Different ways to specify what you're looking for.

Full description

Beautiful Soup accepts various filter types: strings for exact matches, regular expressions for patterns, lists for multiple options, True for any tag, and custom functions for complex logic.

A string

soup.find_all("tag_name")
soup.find("tag_name")
Example: String filter
links = soup.find_all("a")
print(f"Found {len(links)} links")
for link in links:
    print(f"  - {link.string}")
Found 3 links - Elsie - Lacie - Tillie

A regular expression

import re
soup.find_all(re.compile("pattern"))
soup.find(re.compile("pattern"))
Example: Regex filter
import re

# Find all tags starting with 'b'
tags = soup.find_all(re.compile("^b"))
print("Tags starting with 'b':")
for tag in tags:
    print(f"  - {tag.name}")
Tags starting with 'b': - body - b

A list

soup.find_all([tag1, tag2, ...])
soup.find([tag1, tag2, ...])
Example: List filter

💡 Tips & Best Practices

Performance Tips

  • Use lxml parser for faster parsing: BeautifulSoup(html, 'lxml')
  • Use limit parameter if you only need first few results: find_all('a', limit=10)
  • Use CSS selectors for complex queries: soup.select('div.main > p.intro')
  • Compile regex patterns outside loops: pattern = re.compile('example')

Common Pitfalls

  • .string returns None if tag has multiple children - use .get_text() instead
  • Remember: class is reserved in Python - use class_ parameter
  • Check if attribute exists before accessing: tag.get('href') instead of tag['href']
  • Be careful with .next_sibling - might be whitespace, use .find_next_sibling()

Debugging Tips

  • Print soup structure: print(soup.prettify())
  • Check tag name: print(tag.name)
  • View all attributes: print(tag.attrs)
  • Inspect type: print(type(tag))
  • Test selectors incrementally: start simple, add complexity
