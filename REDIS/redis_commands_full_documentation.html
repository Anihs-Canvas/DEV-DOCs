Hash Commands

Redis hashes are maps between string fields and string values, making them perfect for representing objects like our Django model instances. In this section, we'll explore all hash commands with practical examples using our JobPost models.

HSET

Docs: Redis HSET Documentation

Creates or modifies the value of a field in a hash.

Full description

HSET is the fundamental command for storing data in Redis hashes. It sets field-value pairs in a hash stored at key. If the hash doesn't exist, it creates a new one. If the field already exists, it overwrites the old value. This is perfect for caching Django model instances where each field represents a model attribute.

Syntax

HSET key field value [field value ...]

Parameters

  • key: The name of the hash (string). Best practice: use a namespace like "jobpost:1" or "author:user1".
  • field: The field name within the hash (string). Use model field names like "title", "salary", "company".
  • value: The value to store (string). All Redis values are strings, so serialize complex data as JSON.
  • [field value ...]: Optional additional field-value pairs. You can set multiple fields in one command for efficiency.

Return value

Integer: The number of fields that were added (new fields only, not updated fields).

5 short examples with Django models

Example 1: Cache a JobPost instance
import redis
from jobs.models import JobPost

r = redis.Redis(host='localhost', port=6379, db=0, decode_responses=True)

# Get a JobPost and cache it
job = JobPost.objects.first()
result = r.hset(f"jobpost:{job.id}", mapping={
    "title": job.title,
    "description": job.description,
    "salary": str(job.salary),
    "type": job.type
})
print(result)
4
Example 2: Cache Author with company info
from jobs.models import Author

author = Author.objects.first()
result = r.hset(f"author:{author.id}", mapping={
    "company": author.company,
    "designation": author.designation,
    "username": author.user.username
})
print(result)
3
Example 3: Update single field
# Update just the salary field
result = r.hset("jobpost:1", "salary", "95000")
print(result)
0
Example 4: Cache Location data
from jobs.models import Location

location = Location.objects.first()
result = r.hset(f"location:{location.id}", mapping={
                    "city": location.city,
                    "state": location.state,
                    "country": location.country
                })
print(result)
3
Example 5: Cache Skills list as JSON
import json
from jobs.models import JobPost

job = JobPost.objects.prefetch_related('skills').first()
skills_list = list(job.skills.values_list('name', flat=True))
result = r.hset(f"jobpost:{job.id}:skills", "names", json.dumps(skills_list))
print(result)
1

HGET

Docs: Redis HGET Documentation

Returns the value of a field in a hash.

Full description

HGET retrieves the value associated with a specific field in a hash. This is essential for reading cached Django model data. If the field doesn't exist, it returns nil (None in Python). Use this when you need just one field from your cached model instance.

Syntax

HGET key field

Parameters

  • key: The name of the hash containing the field.
  • field: The field name whose value you want to retrieve.

Return value

String: The value associated with the field, or None if the field doesn't exist.

5 short examples with Django models

Example 1: Get job title
title = r.hget("jobpost:1", "title")
print(title)
Python Developer
Example 2: Get author company
company = r.hget("author:1", "company")
print(company)
ABC Corp
Example 3: Get salary as integer
salary = r.hget("jobpost:1", "salary")
print(int(salary) if salary else 0)
95000
Example 4: Get location city
city = r.hget("location:1", "city")
print(city)
New York
Example 5: Get non-existent field
result = r.hget("jobpost:1", "nonexistent")
print(result)
None

HGETALL

Docs: Redis HGETALL Documentation

Returns all fields and values in a hash.

Full description

HGETALL retrieves all field-value pairs from a hash in a single operation. This is extremely useful for reconstructing complete Django model instances from cache. The return value alternates between field names and their values, which redis-py converts to a convenient dictionary.

Syntax

HGETALL key

Parameters

  • key: The name of the hash from which to retrieve all fields and values.

Return value

Dictionary: All field-value pairs in the hash, or empty dict if the hash doesn't exist.

5 short examples with Django models

Example 1: Get complete JobPost data
job_data = r.hgetall("jobpost:1")
print(job_data)
{'title': 'Python Developer', 'description': 'Looking for a skilled Python developer', 'salary': '95000', 'type': 'Full Time'}
Example 2: Get Author info
author_data = r.hgetall("author:1")
print(author_data)
{'company': 'ABC Corp', 'designation': 'Software Engineer', 'username': 'user1'}
Example 3: Get Location details
location_data = r.hgetall("location:1")
print(location_data)
{'city': 'New York', 'state': 'NY', 'country': 'USA'}
Example 4: Reconstruct model-like object
job_data = r.hgetall("jobpost:1")
print(f"Job: {job_data['title']} at ${job_data['salary']}")
Job: Python Developer at $95000
Example 5: Check empty hash
empty_data = r.hgetall("nonexistent:hash")
print(len(empty_data))
0

HDEL

Docs: Redis HDEL Documentation

Deletes one or more fields and their values from a hash.

Full description

HDEL removes specified fields from a hash. If all fields are removed, the hash key itself is automatically deleted. This is useful for invalidating specific cached model fields or cleaning up cache entries when model data changes in Django.

Syntax

HDEL key field [field ...]

Parameters

  • key: The name of the hash containing the fields to delete.
  • field: One or more field names to remove from the hash.

Return value

Integer: The number of fields that were successfully removed (ignores non-existent fields).

5 short examples with Django models

Example 1: Remove salary field
deleted = r.hdel("jobpost:1", "salary")
print(deleted)
1
Example 2: Remove multiple fields
deleted = r.hdel("author:1", "designation", "username")
print(deleted)
2
Example 3: Try to delete non-existent field
deleted = r.hdel("jobpost:1", "nonexistent_field")
print(deleted)
0
Example 4: Clean up location state
deleted = r.hdel("location:1", "state")
print(deleted)
1
Example 5: Remove fields and check remaining
deleted = r.hdel("jobpost:1", "title", "description")
remaining = r.hgetall("jobpost:1")
print(f"Deleted: {deleted}, Remaining: {remaining}")
Deleted: 2, Remaining: {'type': 'Full Time'}

HEXISTS

Docs: Redis HEXISTS Documentation

Determines whether a field exists in a hash.

Full description

HEXISTS checks if a specific field exists within a hash without retrieving its value. This is perfect for cache validation in Django - you can check if specific model fields are cached before attempting to read them, avoiding unnecessary database queries.

Syntax

HEXISTS key field

Parameters

  • key: The name of the hash to check.
  • field: The field name to look for within the hash.

Return value

Integer: 1 if the field exists, 0 if the field or hash doesn't exist.

5 short examples with Django models

Example 1: Check if job title is cached
exists = r.hexists("jobpost:1", "title")
print(bool(exists))
False
Example 2: Validate cache before read
if r.hexists("author:1", "company"):
    company = r.hget("author:1", "company")
    print(f"Cached: {company}")
else:
    print("Not cached")
Cached: ABC Corp
Example 3: Check non-existent field
exists = r.hexists("location:1", "zipcode")
print(bool(exists))
False
Example 4: Conditional caching pattern
key = "jobpost:2"
if not r.hexists(key, "salary"):
    # Cache miss - would fetch from DB
    r.hset(key, "salary", "75000")
    print("Cached from DB")
else:
    print("Already cached")
Cached from DB
Example 5: Check multiple field existence
required_fields = ["company", "designation"]
missing = [f for f in required_fields if not r.hexists("author:1", f)]
print(f"Missing fields: {missing}")
Missing fields: ['designation']

HKEYS

Docs: Redis HKEYS Documentation

Returns all fields in a hash.

Full description

HKEYS returns a list of all field names in a hash without their values. This is useful for inspecting what model attributes are cached, validating cache completeness, or iterating over available fields in your Django model cache.

Syntax

HKEYS key

Parameters

  • key: The name of the hash whose field names you want to retrieve.

Return value

List: All field names in the hash, or empty list if the hash doesn't exist.

5 short examples with Django models

Example 1: Get cached JobPost fields
fields = r.hkeys("jobpost:1")
print(fields)
['type']
Example 2: Check Author cache completeness
cached_fields = r.hkeys("author:1")
required_fields = ["company", "designation", "username"]
complete = all(field in cached_fields for field in required_fields)
print(f"Cache complete: {complete}, Fields: {cached_fields}")
Cache complete: False, Fields: ['company']
Example 3: List all location attributes
location_fields = r.hkeys("location:1")
print(f"Cached location attributes: {location_fields}")
Cached location attributes: ['city', 'country']
Example 4: Count cached fields
field_count = len(r.hkeys("jobpost:2"))
print(f"Number of cached fields: {field_count}")
Number of cached fields: 1
Example 5: Check for empty hash
fields = r.hkeys("nonexistent:hash")
print(f"Fields in non-existent hash: {fields}")
Fields in non-existent hash: []

HVALS

Docs: Redis HVALS Documentation

Returns all values in a hash.

Full description

HVALS returns all values from a hash without their field names. This is useful when you need to process all cached values from a Django model instance, such as checking for empty values or performing data validation across all cached attributes.

Syntax

HVALS key

Parameters

  • key: The name of the hash whose values you want to retrieve.

Return value

List: All values in the hash, or empty list if the hash doesn't exist.

5 short examples with Django models

Example 1: Get all JobPost values
values = r.hvals("jobpost:1")
print(values)
['Full Time']
Example 2: Check for empty values
values = r.hvals("author:1")
has_empty = any(not v.strip() for v in values if v)
print(f"Has empty values: {has_empty}, Values: {values}")
Has empty values: False, Values: ['ABC Corp']
Example 3: Get location values for validation
location_values = r.hvals("location:1")
print(f"Location data: {location_values}")
Location data: ['New York', 'USA']
Example 4: Count non-empty values
values = r.hvals("jobpost:2")
non_empty_count = len([v for v in values if v and v.strip()])
print(f"Non-empty values: {non_empty_count}")
Non-empty values: 1
Example 5: Process all values
values = r.hvals("author:1")
total_length = sum(len(str(v)) for v in values)
print(f"Total character count: {total_length}")
Total character count: 8

HLEN

Docs: Redis HLEN Documentation

Returns the number of fields in a hash.

Full description

HLEN returns the count of fields in a hash. This is perfect for validating cache completeness in Django - you can check if all required model fields are cached by comparing the field count with expected values.

Syntax

HLEN key

Parameters

  • key: The name of the hash whose field count you want to retrieve.

Return value

Integer: The number of fields in the hash, or 0 if the hash doesn't exist.

5 short examples with Django models

Example 1: Check JobPost cache completeness
field_count = r.hlen("jobpost:1")
expected_fields = 4  # title, description, salary, type
is_complete = field_count == expected_fields
print(f"Fields cached: {field_count}, Complete: {is_complete}")
Fields cached: 1, Complete: False
Example 2: Validate Author cache
author_field_count = r.hlen("author:1")
print(f"Author fields cached: {author_field_count}")
Author fields cached: 1
Example 3: Check empty hash
empty_count = r.hlen("nonexistent:hash")
print(f"Fields in non-existent hash: {empty_count}")
Fields in non-existent hash: 0
Example 4: Monitor cache growth
# Add a field and check count
r.hset("location:1", "zip", "10001")
new_count = r.hlen("location:1")
print(f"Location fields after adding zip: {new_count}")
Location fields after adding zip: 3
Example 5: Cache efficiency check
counts = {
            "jobpost": r.hlen("jobpost:1"),
            "author": r.hlen("author:1"),
            "location": r.hlen("location:1")
        }
print(f"Cache field counts: {counts}")
Cache field counts: {'jobpost': 1, 'author': 1, 'location': 3}

HMGET

Docs: Redis HMGET Documentation

Returns the values of multiple fields in a hash.

Full description

HMGET retrieves values for multiple specified fields in a single operation. This is more efficient than multiple HGET calls and perfect for fetching specific Django model attributes you need for rendering views or API responses.

Syntax

HMGET key field [field ...]

Parameters

  • key: The name of the hash containing the fields.
  • field: One or more field names whose values you want to retrieve.

Return value

List: Values for the requested fields in the same order, with None for non-existent fields.

5 short examples with Django models

Example 1: Get job title and type
# First cache some data
r.hset("jobpost:3", mapping={"title": "Frontend Developer", "type": "Part Time", "salary": "65000"})
title, job_type = r.hmget("jobpost:3", "title", "type")
print(f"Title: {title}, Type: {job_type}")
Title: Frontend Developer, Type: Part Time
Example 2: Get author details for profile
# Cache author data first
r.hset("author:2", mapping={"company": "XYZ Inc", "designation": "Web Developer", "username": "user2"})
company, designation = r.hmget("author:2", "company", "designation")
print(f"{designation} at {company}")
Web Developer at XYZ Inc
Example 3: Get location for display
city, country, zip_code = r.hmget("location:1", "city", "country", "zip")
location_str = f"{city}, {country} {zip_code or ''}"
print(location_str.strip())
New York, USA 10001
Example 4: Handle missing fields
values = r.hmget("jobpost:3", "title", "missing_field", "salary")
filtered_values = [v for v in values if v is not None]
print(f"Available values: {filtered_values}")
Available values: ['Frontend Developer', '65000']
Example 5: Build model-like dict
fields = ["title", "type", "salary"]
values = r.hmget("jobpost:3", *fields)
job_dict = dict(zip(fields, values))
print(job_dict)
{'title': 'Frontend Developer', 'type': 'Part Time', 'salary': '65000'}

HINCRBY

Docs: Redis HINCRBY Documentation

Increments the integer value of a field in a hash by a number.

Full description

HINCRBY atomically increments a hash field by an integer amount. If the field doesn't exist, it starts from 0. This is perfect for counters in Django applications - tracking job views, application counts, or any numeric metrics associated with your models.

Syntax

HINCRBY key field increment

Parameters

  • key: The name of the hash containing the field to increment.
  • field: The field name whose value should be incremented.
  • increment: The integer amount to add (can be negative for decrement).

Return value

Integer: The new value after incrementing.

5 short examples with Django models

Example 1: Track job view count
views = r.hincrby("jobpost:1:stats", "view_count", 1)
print(f"Job views: {views}")
Job views: 1
Example 2: Count applications per job
applications = r.hincrby("jobpost:2:stats", "applications", 1)
print(f"Total applications: {applications}")
Total applications: 1
Example 3: Track author job posts
job_count = r.hincrby("author:1:stats", "jobs_posted", 1)
print(f"Jobs posted by author: {job_count}")
Jobs posted by author: 1
Example 4: Decrement salary budget
# Set initial budget
r.hset("company:abc:budget", "remaining", "100000")
remaining = r.hincrby("company:abc:budget", "remaining", -80000)
print(f"Remaining budget: ${remaining}")
Remaining budget: $20000
Example 5: Track location job counts
job_count = r.hincrby("location:1:stats", "active_jobs", 1)
print(f"Active jobs in location: {job_count}")
Active jobs in location: 1

HSETNX

Docs: Redis HSETNX Documentation

Sets the value of a field in a hash only when the field doesn't exist.

Full description

HSETNX sets a field only if it doesn't already exist in the hash. This is perfect for initializing default values in Django model caches, ensuring you don't overwrite existing data, or implementing cache-based locks and unique constraints.

Syntax

HSETNX key field value

Parameters

  • key: The name of the hash to set the field in.
  • field: The field name to set (only if it doesn't exist).
  • value: The value to set for the field.

Return value

Integer: 1 if the field was set (didn't exist), 0 if the field already exists.

5 short examples with Django models

Example 1: Set default job status
was_set = r.hsetnx("jobpost:4", "status", "active")
print(f"Status set: {bool(was_set)}")
Status set: True
Example 2: Try to overwrite existing field
was_set = r.hsetnx("jobpost:4", "status", "inactive")
current_status = r.hget("jobpost:4", "status")
print(f"Overwrite successful: {bool(was_set)}, Current: {current_status}")
Overwrite successful: False, Current: active
Example 3: Initialize author metrics
metrics_set = r.hsetnx("author:3:metrics", "rating", "5.0")
print(f"Initial rating set: {bool(metrics_set)}")
Initial rating set: True
Example 4: Safe default location setting
was_set = r.hsetnx("location:5", "timezone", "UTC")
timezone = r.hget("location:5", "timezone")
print(f"Timezone set: {bool(was_set)}, Value: {timezone}")
Timezone set: True, Value: UTC
Example 5: Conditional cache initialization
fields_to_init = {"created_at": "2025-08-24", "updated_at": "2025-08-24"}
results = []
for field, value in fields_to_init.items():
    result = r.hsetnx("jobpost:5", field, value)
    results.append(f"{field}: {bool(result)}")
print(f"Initialization results: {results}")
Initialization results: ['created_at: True', 'updated_at: True']

HSTRLEN

Docs: Redis HSTRLEN Documentation

Returns the length of the value of a field.

Full description

HSTRLEN returns the string length of a hash field's value. This is useful for validating Django model field lengths, checking description sizes, or implementing character limits in your cached model data.

Syntax

HSTRLEN key field

Parameters

  • key: The name of the hash containing the field.
  • field: The field whose value length you want to measure.

Return value

Integer: The length of the field's value, or 0 if the field doesn't exist.

5 short examples with Django models

Example 1: Check job title length
r.hset("jobpost:6", "title", "Senior Python Developer")
title_length = r.hstrlen("jobpost:6", "title")
print(f"Title length: {title_length} characters")
Title length: 23 characters
Example 2: Validate description length
r.hset("jobpost:6", "description", "We are looking for an experienced Python developer")
desc_length = r.hstrlen("jobpost:6", "description")
is_valid = desc_length <= 200
print(f"Description: {desc_length} chars, Valid: {is_valid}")
Description: 45 chars, Valid: True
Example 3: Check company name length
r.hset("author:4", "company", "Tech Innovations LLC")
company_length = r.hstrlen("author:4", "company")
print(f"Company name: {company_length} characters")
Company name: 20 characters
Example 4: Check non-existent field
missing_length = r.hstrlen("jobpost:6", "nonexistent")
print(f"Non-existent field length: {missing_length}")
Non-existent field length: 0
Example 5: Validate all field lengths
fields = ["title", "description"]
lengths = {field: r.hstrlen("jobpost:6", field) for field in fields}
total_chars = sum(lengths.values())
print(f"Field lengths: {lengths}, Total: {total_chars}")
Field lengths: {'title': 23, 'description': 45}, Total: 68
