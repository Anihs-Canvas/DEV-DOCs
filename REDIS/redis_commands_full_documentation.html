HSET

Docs: Redis HSET Documentation

Creates or modifies the value of a field in a hash.

Full description

HSET is the fundamental command for storing data in Redis hashes. It sets field-value pairs in a hash stored at key. If the hash doesn't exist, it creates a new one. If the field already exists, it overwrites the old value. This is perfect for caching Django model instances where each field represents a model attribute.

Syntax

HSET key field value [field value ...]

Parameters

  • key: The name of the hash (string). Best practice: use a namespace like "jobpost:1" or "author:user1".
  • field: The field name within the hash (string). Use model field names like "title", "salary", "company".
  • value: The value to store (string). All Redis values are strings, so serialize complex data as JSON.
  • [field value ...]: Optional additional field-value pairs. You can set multiple fields in one command for efficiency.

Return value

Integer: The number of fields that were added (new fields only, not updated fields).

5 short examples with Django models

Example 1: Cache a JobPost instance
import redis
from jobs.models import JobPost

r = redis.Redis(host='localhost', port=6379, db=0, decode_responses=True)

# Get a JobPost and cache it
job = JobPost.objects.first()
result = r.hset(f"jobpost:{job.id}", mapping={
    "title": job.title,
    "description": job.description,
    "salary": str(job.salary),
    "type": job.type
})
print(result)
4
Example 2: Cache Author with company info
from jobs.models import Author

author = Author.objects.first()
result = r.hset(f"author:{author.id}", mapping={
    "company": author.company,
    "designation": author.designation,
    "username": author.user.username
})
print(result)
3
Example 3: Update single field
# Update just the salary field
result = r.hset("jobpost:1", "salary", "95000")
print(result)
0
Example 4: Cache Location data
from jobs.models import Location

location = Location.objects.first()
result = r.hset(f"location:{location.id}", mapping={
                    "city": location.city,
                    "state": location.state,
                    "country": location.country
                })
print(result)
3
Example 5: Cache Skills list as JSON
import json
from jobs.models import JobPost

job = JobPost.objects.prefetch_related('skills').first()
skills_list = list(job.skills.values_list('name', flat=True))
result = r.hset(f"jobpost:{job.id}:skills", "names", json.dumps(skills_list))
print(result)
1

HGET

Docs: Redis HGET Documentation

Returns the value of a field in a hash.

Full description

HGET retrieves the value associated with a specific field in a hash. This is essential for reading cached Django model data. If the field doesn't exist, it returns nil (None in Python). Use this when you need just one field from your cached model instance.

Syntax

HGET key field

Parameters

  • key: The name of the hash containing the field.
  • field: The field name whose value you want to retrieve.

Return value

String: The value associated with the field, or None if the field doesn't exist.

5 short examples with Django models

Example 1: Get job title
title = r.hget("jobpost:1", "title")
print(title)
Python Developer
Example 2: Get author company
company = r.hget("author:1", "company")
print(company)
ABC Corp
Example 3: Get salary as integer
salary = r.hget("jobpost:1", "salary")
print(int(salary) if salary else 0)
95000
Example 4: Get location city
city = r.hget("location:1", "city")
print(city)
New York
Example 5: Get non-existent field
result = r.hget("jobpost:1", "nonexistent")
print(result)
None

HGETALL

Docs: Redis HGETALL Documentation

Returns all fields and values in a hash.

Full description

HGETALL retrieves all field-value pairs from a hash in a single operation. This is extremely useful for reconstructing complete Django model instances from cache. The return value alternates between field names and their values, which redis-py converts to a convenient dictionary.

Syntax

HGETALL key

Parameters

  • key: The name of the hash from which to retrieve all fields and values.

Return value

Dictionary: All field-value pairs in the hash, or empty dict if the hash doesn't exist.

5 short examples with Django models

Example 1: Get complete JobPost data
job_data = r.hgetall("jobpost:1")
print(job_data)
{'title': 'Python Developer', 'description': 'Looking for a skilled Python developer', 'salary': '95000', 'type': 'Full Time'}
Example 2: Get Author info
author_data = r.hgetall("author:1")
print(author_data)
{'company': 'ABC Corp', 'designation': 'Software Engineer', 'username': 'user1'}
Example 3: Get Location details
location_data = r.hgetall("location:1")
print(location_data)
{'city': 'New York', 'state': 'NY', 'country': 'USA'}
Example 4: Reconstruct model-like object
job_data = r.hgetall("jobpost:1")
print(f"Job: {job_data['title']} at ${job_data['salary']}")
Job: Python Developer at $95000
Example 5: Check empty hash
empty_data = r.hgetall("nonexistent:hash")
print(len(empty_data))

Deletes one or more fields and their values from a hash.

Full description

HDEL removes specified fields from a hash. If all fields are removed, the hash key itself is automatically deleted. This is useful for invalidating specific cached model fields or cleaning up cache entries when model data changes in Django.

deleted = r.hdel("jobpost:1", "salary")

HEXISTS

Docs: Redis HEXISTS Documentation

Determines whether a field exists in a hash.

Full description

HEXISTS checks if a specific field exists within a hash without retrieving its value. This is perfect for cache validation in Django - you can check if specific model fields are cached before attempting to read them, avoiding unnecessary database queries.

Syntax

HEXISTS key field

Parameters

  • key: The name of the hash to check.
  • field: The field name to look for within the hash.

Return value

Integer: 1 if the field exists, 0 if the field or hash doesn't exist.

5 short examples with Django models

Example 1: Check if job title is cached
exists = r.hexists("jobpost:1", "title")
print(bool(exists))
False
Example 2: Validate cache before read
if r.hexists("author:1", "company"):
    company = r.hget("author:1", "company")
    print(f"Cached: {company}")
else:
    print("Not cached")
Cached: ABC Corp
Example 3: Check non-existent field
exists = r.hexists("location:1", "zipcode")
print(bool(exists))
False
Example 4: Conditional caching pattern
key = "jobpost:2"
if not r.hexists(key, "salary"):
    # Cache miss - would fetch from DB
    r.hset(key, "salary", "75000")
    print("Cached from DB")
else:
    print("Already cached")
Cached from DB
Example 5: Check multiple field existence
required_fields = ["company", "designation"]
missing = [f for f in required_fields if not r.hexists("author:1", f)]
print(f"Missing fields: {missing}")
Missing fields: ['designation']

HMGET

Docs: Redis HMGET Documentation

Returns the values of multiple fields in a hash.

Full description

HMGET retrieves values for multiple specified fields in a single operation. This is more efficient than multiple HGET calls and perfect for fetching specific Django model attributes you need for rendering views or API responses.

Syntax

HMGET key field [field ...]

Parameters

  • key: The name of the hash containing the fields.
  • field: One or more field names whose values you want to retrieve.

Return value

List: Values for the requested fields in the same order, with None for non-existent fields.

5 short examples with Django models

Example 1: Get job title and type
# First cache some data
r.hset("jobpost:3", mapping={"title": "Frontend Developer", "type": "Part Time", "salary": "65000"})
title, job_type = r.hmget("jobpost:3", "title", "type")
print(f"Title: {title}, Type: {job_type}")
Title: Frontend Developer, Type: Part Time
Example 2: Get author details for profile
# Cache author data first
r.hset("author:2", mapping={"company": "XYZ Inc", "designation": "Web Developer", "username": "user2"})
company, designation = r.hmget("author:2", "company", "designation")
print(f"{designation} at {company}")
Web Developer at XYZ Inc
Example 3: Get location for display
city, country, zip_code = r.hmget("location:1", "city", "country", "zip")
location_str = f"{city}, {country} {zip_code or ''}"
print(location_str.strip())
New York, USA 10001
Example 4: Handle missing fields
values = r.hmget("jobpost:3", "title", "missing_field", "salary")
filtered_values = [v for v in values if v is not None]
print(f"Available values: {filtered_values}")
Available values: ['Frontend Developer', '65000']
Example 5: Build model-like dict
fields = ["title", "type", "salary"]
values = r.hmget("jobpost:3", *fields)
job_dict = dict(zip(fields, values))
print(job_dict)
{'title': 'Frontend Developer', 'type': 'Part Time', 'salary': '65000'}

HSETNX sets a field only if it doesn't already exist in the hash. This is perfect for initializing default values in Django model caches, ensuring you don't overwrite existing data, or implementing cache-based locks and unique constraints.

Parameters

Integer: 1 if the field was set (didn't exist), 0 if the field already exists.

was_set = r.hsetnx("jobpost:4", "status", "active")

was_set = r.hsetnx("jobpost:4", "status", "inactive")

metrics_set = r.hsetnx("author:3:metrics", "rating", "5.0")
print(f"Initial rating set: {bool(metrics_set)}")
Initial rating set: True
Example 4: Safe default location setting
was_set = r.hsetnx("location:5", "timezone", "UTC")

fields_to_init = {"created_at": "2025-08-24", "updated_at": "2025-08-24"}

Initialization results: ['created_at: True', 'updated_at: True']

SET

Docs: Redis SET Documentation

Sets the string value of a key, ignoring its type. The key is created if it doesn't exist.

Full description

SET is the fundamental Redis command for storing string values. It creates a new key or overwrites an existing one regardless of its previous type. In Django applications, use SET for caching serialized model data, storing session information, feature flags, or any simple key-value pairs. The command supports various options for expiration and conditional setting.

Syntax

SET key value [EX seconds|PX milliseconds|EXAT unix-timestamp|PXAT unix-timestamp-ms] [NX|XX] [GET]

Parameters

  • key: The name of the key to set (string). Use descriptive names like "user:1:email" or "job:5:status".
  • value: The string value to store. Can be text, numbers, JSON, or serialized data.

  • PX milliseconds: Set expiration in milliseconds for fine-grained control.

Bitmap Commands

Redis Bitmaps are space-efficient binary arrays that enable fast bit-level operations on large datasets. Perfect for Django applications requiring efficient tracking of binary states - like user activity patterns, feature flags, permissions, online/offline status, or analytics where you need to track millions of boolean values with minimal memory usage. Each bit represents a boolean state, making bitmaps ideal for real-time analytics and high-performance tracking systems.

🔢 Bitmap Commands Overview

BITCOUNT - Count set bits in range
BITFIELD - Multi-bit integer operations
BITFIELD_RO - Read-only bitfield operations
BITOP - Bitwise operations (AND/OR/XOR)
BITPOS - Find first set/clear bit
GETBIT - Get single bit value
SETBIT - Set single bit value
💡 Pro Tip: Bitmaps can represent up to 4 billion boolean values in just 512MB of memory, making them incredibly efficient for large-scale user tracking and analytics in Django applications.

Cuckoo Filter Commands

Redis Cuckoo Filters are space-efficient probabilistic data structures that support both membership testing and deletion operations. Unlike Bloom Filters, Cuckoo Filters allow item removal while maintaining excellent performance and low false positive rates. Perfect for Django applications requiring dynamic sets with add/remove operations - like active user sessions, temporary access tokens, recent activity tracking, or any scenario where you need both fast lookups and the ability to clean up expired items.

🥒 Cuckoo Filter Commands Overview

CF.ADD - Add single item to filter
CF.ADDNX - Add item if not exists
CF.COUNT - Count item occurrences
CF.DEL - Delete item from filter
CF.EXISTS - Check item existence
CF.INFO - Get filter information
CF.INSERT - Batch insert items
CF.INSERTNX - Batch insert if not exists
CF.LOADCHUNK - Restore filter chunk
CF.MEXISTS - Check multiple items
CF.RESERVE - Create new filter
CF.SCANDUMP - Dump filter for backup
🔄 Key Advantage: Unlike Bloom Filters, Cuckoo Filters support deletion operations, making them ideal for dynamic datasets where items need to be removed (like session management, token expiration, or cache invalidation).

CF.RESERVE creates a new Cuckoo Filter with specified capacity and optional bucket size and maximum iterations. RPOP

Docs: Redis RPOP Documentation

Remove and return the last element of a list.

Full description

RPOP removes and returns the last element from the right (tail) of a list. Combined with RPUSH, this creates perfect FIFO queue behavior. In Django applications, this is ideal for processing tasks in order, handling job queues, or implementing any workflow where items should be processed in the sequence they were added.

Syntax

RPOP key [count]

Parameters

  • key: The name of the list.
  • count: Optional. Number of elements to pop (Redis 6.2+).

Return value

String: The popped element, or nil if the list is empty. If count is specified, returns an array.

def get_paginated_jobs(user_id, page=1, per_page=5):
    user_jobs_key = f"user:{user_id}:job_posts"
    total_jobs = r.llen(user_jobs_key)
    
    if total_jobs == 0:
        return [], {"total": 0, "pages": 0, "current_page": 0}
    
    total_pages = (total_jobs - 1) // per_page + 1
    start_index = (page - 1) * per_page
    end_index = start_index + per_page - 1
    
    # Get the specific page of jobs
    jobs = r.lrange(user_jobs_key, start_index, end_index)
    
    pagination_info = {
        "total": total_jobs,
        "pages": total_pages,
        "current_page": page,
        "per_page": per_page,
        "has_next": page < total_pages,
        "has_prev": page > 1
    }
    
    return jobs, pagination_info

HyperLogLog Commands

Redis HyperLogLog is a probabilistic data structure used to estimate the cardinality (number of unique elements) in very large datasets with remarkable space efficiency. Perfect for Django applications that need to track unique visitors, count distinct user actions, or estimate unique values in massive datasets without storing each individual element.

📊 HyperLogLog Commands Overview

PFADD - Add elements to HyperLogLog
PFCOUNT - Get cardinality estimate
PFMERGE - Merge multiple HyperLogLogs
PFDEBUG - Debug HyperLogLog internals
PFSELFTEST - Run HyperLogLog self-test
🎯 Perfect for Django: HyperLogLog excels at counting unique visitors, tracking distinct user interactions, estimating unique job views, or measuring distinct skill combinations - all with minimal memory usage even for millions of unique values.

PFADD

Docs: Redis PFADD Documentation

Add elements to a HyperLogLog data structure.

Full description

PFADD adds one or more elements to the HyperLogLog data structure. The HyperLogLog will then be able to estimate the cardinality of the set of elements added so far. This is perfect for tracking unique visitors, distinct user actions, or any scenario where you need to count unique items efficiently without storing each individual element.

Syntax

PFADD key element [element ...]

Parameters

  • key: The HyperLogLog key name.
  • element: The element(s) to add to the HyperLogLog.

Return value

Integer: 1 if the internal state was modified, 0 if the approximated cardinality didn't change.
