🏗️ Workload Resources

Workload resources are the core of Kubernetes, managing how your Django applications run. These resources control the lifecycle, scaling, and scheduling of your containerized applications across the cluster.

💡 Django Best Practice: Use Pods directly for development and testing. In production, use Deployments to manage Pod replicas with rolling updates and high availability.

Pod

Docs: Kubernetes Pod Documentation

Pod is a collection of containers that can run on a host. This resource is created by clients and scheduled onto hosts.

Full description

Pod is the basic deployable unit in Kubernetes. For Django applications, a Pod typically contains your Django container along with any sidecar containers like log collectors or monitoring agents.

Example Django Pod Configuration

Basic Django Pod
apiVersion: v1
kind: Pod
metadata:
  name: django-web
  labels:
    app: django-web
    tier: frontend
spec:
  containers:
  - name: django-app
    image: your-django-app:latest
    ports:
    - containerPort: 8000
    env:
    - name: DJANGO_SETTINGS_MODULE
      value: "myproject.settings"
    - name: DATABASE_URL
      value: "postgresql://user:pass@db:5432/mydb"
kubectl output:
pod/django-web created
