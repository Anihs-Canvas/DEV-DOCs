🗄️ Interacting with etcd (v3 API)

A simplified reference for using etcdctl to manage an etcd cluster.

etcd is a distributed, reliable key-value store. Most interactions involve putting (writing) and getting (reading) keys. This guide uses etcdctl, the official command-line tool. The concepts apply equally to gRPC and client-library APIs.

By default, etcdctl on master (v3.4+) uses the v3 API. Older versions (≤ 3.3) default to the v2 API. Keys created with the v2 API cannot be queried with the v3 API.

1. Checking the Version

Find the client and server API versions to ensure you are using the correct commands.

$ etcdctl version
etcdctl version: 3.1.0-alpha.0+git
API version: 3.1

2. Writing a Key

Every key stored in etcd is replicated to all cluster members via the Raft protocol, guaranteeing consistency and reliability.

Simple Put

$ etcdctl put foo bar
OK

Put with a Lease (Time‑to‑Live)

Attach a lease to a key so that it automatically expires after a set duration. First grant a lease, then use its ID.

$ etcdctl put foo1 bar1 --lease=1234abcd
OK

⏳ The lease ID (1234abcd) is returned when you create a lease. The key will be deleted when the lease expires.

3. Reading Keys

Read a single key, a range of keys, or filter by prefix.

Get a Single Key

$ etcdctl get foo
foo
bar

Get Value Only (No Key Name)

$ etcdctl get foo --print-value-only
bar

Get in Hexadecimal Format

$ etcdctl get foo --hex
\x66\x6f\x6f          # Key
\x62\x61\x72          # Value

Range Query (Half‑Open Interval)

The range [foo, foo3) includes foo and foo2 but excludes foo3.

$ etcdctl get foo foo3
foo
bar
foo1
bar1
foo2
bar2

All Keys with a Prefix

$ etcdctl get --prefix foo
foo
bar
foo1
bar1
foo2
bar2
foo3
bar3

Prefix Query with a Limit

$ etcdctl get --prefix --limit=2 foo
foo
bar
foo1
bar1

Keys Greater Than or Equal to a Byte Value

$ etcdctl get --from-key b
b
456
z
789

4. Reading Past Versions (History)

Every modification increments the cluster’s global revision. You can read keys as they were at a specific revision—useful for rollbacks or consistent snapshots.

Example history:

# Revision 2: foo = bar
# Revision 3: foo1 = bar1
# Revision 4: foo = bar_new
# Revision 5: foo1 = bar1_new
$ etcdctl get --prefix foo
foo
bar_new
foo1
bar1_new
(most recent versions)
$ etcdctl get --prefix --rev=4 foo
foo
bar_new
foo1
bar1
(versions at revision 4)
$ etcdctl get --prefix --rev=3 foo
foo
bar
foo1
bar1
(versions at revision 3)
$ etcdctl get --prefix --rev=2 foo
foo
bar
(versions at revision 2)

5. Deleting Keys

Delete a single key, a range, or by prefix.

Delete a Single Key

$ etcdctl del foo
1
(one key deleted)

Delete a Range

$ etcdctl del foo foo9
2
(two keys deleted)

Delete and Show the Deleted Key‑Value

$ etcdctl del --prev-kv zoo
1
zoo
val
(one key deleted; key and value printed)

Delete by Prefix

$ etcdctl del --prefix zoo
2
(two keys deleted)

Delete Keys Greater Than or Equal to a Byte Value

$ etcdctl del --from-key b
2
(two keys deleted)

6. Watching for Changes

Monitor a key or range of keys in real time. The watch command runs continuously until you stop it (Ctrl+C).

Watch a Single Key

# Terminal 1:
$ etcdctl watch foo
# Terminal 2:
$ etcdctl put foo bar
PUT
foo
bar

Watch in Hex Format

# Terminal 1:
$ etcdctl watch foo --hex
# Terminal 2:
$ etcdctl put foo bar
PUT
\x66\x6f\x6f          # Key
\x62\x61\x72          # Value

Watch a Range

# Terminal 1:
$ etcdctl watch foo foo9
# Terminal 2:
$ etcdctl put foo bar
PUT
foo
bar
# Terminal 2:
$ etcdctl put foo1 bar1
PUT
foo1
bar1

Watch by Prefix

# Terminal 1:
$ etcdctl watch --prefix foo
# Terminal 2:
$ etcdctl put foo bar
PUT
foo
bar
# Terminal 2:
$ etcdctl put fooz1 barz1
PUT
fooz1
barz1

Interactive Watch (Multiple Keys)

Start an interactive session and issue multiple watch commands.

$ etcdctl watch -i
$ watch foo
$ watch zoo
# Terminal 2:
$ etcdctl put foo bar
PUT
foo
bar
# Terminal 2:
$ etcdctl put zoo val
PUT
zoo
val

7. Watching Historical Changes

If your application disconnects, you may miss events. Use --rev to replay changes from a specific revision.

Assume the following sequence:

$ etcdctl put foo bar        # revision 2
$ etcdctl put foo1 bar1      # revision 3
$ etcdctl put foo bar_new    # revision 4
$ etcdctl put foo1 bar1_new  # revision 5

Watch Since Revision 2

$ etcdctl watch --rev=2 foo
PUT
foo
bar
PUT
foo
bar_new

Watch Since Revision 3

$ etcdctl watch --rev=3 foo
PUT
foo
bar_new

Watch with Previous Key‑Value (--prev-kv)

# Terminal 1:
$ etcdctl watch --prev-kv foo
# Terminal 2:
$ etcdctl put foo bar_latest
PUT
foo          # key
bar_new      # previous value
foo          # key
bar_latest   # new value

8. Watch Progress

Check how up‑to‑date a watch stream is. Useful for cache invalidation.

$ etcdctl watch -i
$ watch a
$ progress
progress notify: 1
(no events yet)

# In another terminal:
$ etcdctl put x 0
$ etcdctl put y 1

$ progress
progress notify: 3

⚠️ The revision in the progress notification comes from the local etcd node the watch is connected to. If that node is partitioned, the revision may be lower than the quorum’s true revision.

9. Compacting Revisions

etcd keeps historical revisions so you can travel back in time. To prevent unbounded storage growth, you must compact old revisions periodically. After compaction, data before the compacted revision is permanently removed.

Compact Up to Revision 5

$ etcdctl compact 5
compacted revision 5

Attempt to Read a Compacted Revision

$ etcdctl get --rev=4 foo
Error: rpc error: code = 11 desc = etcdserver: mvcc: required revision has been compacted

Find the Current Revision

Query any key (even a non‑existent one) in JSON format to see the current revision.

$ etcdctl get mykey -w=json
{"header":{"cluster_id":14841639068965178418,"member_id":10276657743932975437,"revision":15,"raft_term":4}}

10. Working with Leases

Leases bind a time‑to‑live (TTL) to keys. When a lease expires, all attached keys are deleted automatically.

Grant a Lease

Create a lease with a 60‑second TTL.

$ etcdctl lease grant 60
lease 32695410dcc0ca06 granted with TTL(60s)

Attach a Key to the Lease

$ etcdctl put --lease=32695410dcc0ca06 foo bar
OK

Revoke a Lease (Deletes All Attached Keys)

$ etcdctl lease revoke 32695410dcc0ca06
lease 32695410dcc0ca06 revoked

$ etcdctl get foo
(empty response – key was deleted)

Keep a Lease Alive (Refresh TTL)

Prevent a lease from expiring by sending periodic keep‑alive requests.

$ etcdctl lease keep-alive 32695410dcc0ca06
lease 32695410dcc0ca06 keepalived with TTL(60)
lease 32695410dcc0ca06 keepalived with TTL(60)
lease 32695410dcc0ca06 keepalived with TTL(60)
...

Get Lease Information

$ etcdctl lease grant 500
lease 694d5765fc71500b granted with TTL(500s)

$ etcdctl put zoo1 val1 --lease=694d5765fc71500b
OK

$ etcdctl put zoo2 val2 --lease=694d5765fc71500b
OK

$ etcdctl lease timetolive 694d5765fc71500b
lease 694d5765fc71500b granted with TTL(500s), remaining(258s)

$ etcdctl lease timetolive --keys 694d5765fc71500b
lease 694d5765fc71500b granted with TTL(500s), remaining(132s), attached keys([zoo2 zoo1])

If the lease has expired or does not exist:

$ etcdctl lease timetolive 694d5765fc71500b
Error: etcdserver: requested lease not found
