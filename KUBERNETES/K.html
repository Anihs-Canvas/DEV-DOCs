ReplicationController

Docs: Kubernetes ReplicationController v1

ReplicationController represents the configuration of a replication controller.

📋 ReplicationController Components Overview

A ReplicationController ensures that a specified number of pod replicas are running at any one time. In other words, a ReplicationController makes sure that a pod or a homogeneous set of pods is always up and available. If there are too many pods, the ReplicationController terminates the extra pods. If there are too few, the ReplicationController starts more pods.

For Django applications, ReplicationControllers provide basic scaling and self-healing capabilities. However, modern Kubernetes deployments typically use Deployment resources instead, which provide more advanced features like rolling updates and rollback capabilities.

💡 Note: While ReplicationControllers are still supported, it's recommended to use Deployments for new applications as they provide better management capabilities and are the preferred way to ensure your Django applications are properly replicated and updated.

🔧 Core Components

ReplicationControllerSpec Configuration for replica count, selector, and pod template
ReplicationControllerStatus Current state information with replica counts and conditions
ReplicationControllerList Collection management for multiple ReplicationController resources

ReplicationController

Docs: Kubernetes ReplicationController v1 API

Ensures a specified number of Pod replicas are running at any given time, maintaining desired state for Django applications.

Full description

A ReplicationController ensures that a specified number of Pod replicas are running at any one time. For Django applications, this provides basic high availability and load distribution. While largely superseded by Deployments, ReplicationControllers are still useful for simple stateless Django applications where you need guaranteed replica counts. Each replica runs your complete Django application stack, including models like JobPost, Author, Location, and Skills, ensuring your application remains available even if some Pods fail.

Syntax

apiVersion: v1
kind: ReplicationController
metadata:
  name: django-jobpost-rc
  namespace: default
  labels:
    app: django-jobpost
spec:
  replicas: 3
  selector:
    app: django-jobpost
  template:
    metadata:
      labels:
        app: django-jobpost
    spec:
      containers:
      - name: django-app
        image: django-jobpost:latest
        ports:
        - containerPort: 8000

Parameters description

  • apiVersion: Always "v1" for ReplicationController resources. Specifies the Kubernetes API version to use for this resource.
  • kind: Must be "ReplicationController" to indicate this is a ReplicationController resource definition.
  • metadata: Standard object metadata including name, namespace, labels, and annotations. If Labels are empty, they default to the same as the Pod template labels.
  • spec: Specification of the desired behavior of the replication controller. Defines how many replicas to run and the pod template.
  • spec.replicas: Number of desired pod replicas. For Django applications, typically 2-5 replicas for high availability.
  • spec.selector: Label selector for pods managed by this ReplicationController. Must match template labels.
  • spec.template: Pod template that describes the pods that will be created. Contains the Django container specification.
  • status: Most recently observed status of the replication controller. This data may be out of date by some window of time. Populated by the system and read-only.

Return value

ReplicationController object: Returns the created ReplicationController resource with system-assigned fields including current replica count, ready replicas, and creation timestamp. The status will show the progression from 0 replicas to the desired count as Pods are created and become ready.

ReplicationControllerSpec

Docs: Kubernetes ReplicationControllerSpec Documentation

Specification of a ReplicationController defining replica count, pod template, and label selector.

Full description

ReplicationControllerSpec defines the desired behavior of a ReplicationController for Django applications. It includes the replica count, pod template with container specifications, and label selector for managing pods. Essential for maintaining a specified number of Django application replicas to ensure high availability and load distribution.

Parameters description

  • replicas: Number of desired pod replicas. For Django applications, typically 2-5 replicas for high availability and load distribution.
  • selector: Label selector for pods managed by this ReplicationController. Must match the pod template labels for proper pod management.
  • template: Pod template describing the pods that will be created. Contains the Django container specification and resource requirements.
  • minReadySeconds: Minimum number of seconds for which a newly created pod should be ready without crashing for it to be considered available.

Syntax

apiVersion: v1
kind: ReplicationController
metadata:
  name: anihpj-jobpost-rc
  namespace: default
  labels:
    app: anihpj-jobpost
    component: web
spec:
  replicas: 3
  selector:
    app: anihpj-jobpost
    tier: backend
  template:
    metadata:
      labels:
        app: anihpj-jobpost
        tier: backend
    spec:
      containers:
      - name: django-app
        image: anihpj/jobpost:v1.2
        ports:
        - containerPort: 8000
        env:
        - name: DJANGO_SETTINGS_MODULE
          value: "anihpj.settings"
        - name: DATABASE_URL
          value: "postgresql://user:pass@db:5432/anihpj"
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
  minReadySeconds: 5

Example 5: ReplicationController with Node Affinity and Tolerations
apiVersion: v1
kind: ReplicationController
metadata:
  name: anihpj-affinity-rc
  namespace: default
  labels:
    app: anihpj-affinity
    performance: high
spec:
  replicas: 3
  selector:
    app: anihpj-affinity
    performance: high
  template:
    metadata:
      labels:
        app: anihpj-affinity
        performance: high
    spec:
      affinity:
        nodeAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
            nodeSelectorTerms:
            - matchExpressions:
              - key: node-type
                operator: In
                values:
                - high-memory
                - compute-optimized
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 100
            preference:
              matchExpressions:
              - key: zone
                operator: In
                values:
                - us-west-2a
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 100
            podAffinityTerm:
              labelSelector:
                matchExpressions:
                - key: app
                  operator: In
                  values:
                  - anihpj-affinity
              topologyKey: kubernetes.io/hostname
      tolerations:
      - key: "high-performance"
        operator: "Equal"
        value: "true"
        effect: "NoSchedule"
      - key: "dedicated"
        operator: "Exists"
        effect: "NoExecute"
        tolerationSeconds: 300
      containers:
      - name: django-app
        image: anihpj/jobpost:v1.5
        ports:
        - containerPort: 8000
        env:
        - name: PERFORMANCE_MODE
          value: "high"
        - name: NODE_NAME
          valueFrom:
            fieldRef:
              fieldPath: spec.nodeName
        resources:
          requests:
            memory: "1Gi"
            cpu: "1000m"
          limits:
            memory: "2Gi"
            cpu: "2000m"
  minReadySeconds: 25

ReplicationControllerStatus

Docs: Kubernetes ReplicationControllerStatus Documentation

Current status of a ReplicationController including replica counts and condition information.

Full description

ReplicationControllerStatus represents the current state of a ReplicationController providing real-time information about pod replicas and controller conditions. For Django applications, this status helps monitor deployment health, track replica availability, and ensure your JobPost, Author, Location, and Skills services maintain the desired replica count for high availability.

Parameters description

  • replicas: Total number of pod replicas managed by this ReplicationController. Shows the actual count of pods currently running.
  • fullyLabeledReplicas: Number of pods that have all required labels matching the ReplicationController selector.
  • readyReplicas: Number of pods that have a Ready condition. Indicates how many Django application pods are ready to serve traffic.
  • availableReplicas: Number of pods that are available for at least minReadySeconds. Shows pods ready for production traffic.
  • observedGeneration: Most recent generation observed by the ReplicationController. Used to track if the controller has processed the latest changes.
  • conditions: Current service state conditions of the ReplicationController. Provides detailed status information about deployment health.

Syntax

# ReplicationControllerStatus (read-only, populated by Kubernetes)
status:
  replicas: 3
  fullyLabeledReplicas: 3
  readyReplicas: 3
  availableReplicas: 3
  observedGeneration: 1
  conditions:
  - type: ReplicaFailure
    status: "False"
    lastTransitionTime: "2025-09-07T10:30:00Z"
    reason: "ReplicasAvailable"
    message: "All 3 replicas are available and ready"

HTTP API operations for managing ReplicationController resources in Kubernetes clusters.

ReplicationController operations provide HTTP API endpoints for creating, reading, updating, deleting, and listing ReplicationController resources. These operations enable programmatic management of Django application replicas, allowing automated deployment pipelines, monitoring systems, and cluster management tools to interact with your ReplicationController deployments.

GET
HTTP Request
GET /api/v1/namespaces/{namespace}/replicationcontrollers/{name}
Parameters
  • name (string, required, path): Name of the ReplicationController to retrieve.
  • namespace (string, required, path): Namespace containing the ReplicationController.
  • pretty (string, query): If 'true', then the output is pretty printed for better readability.
Return Value

200 (ReplicationController): OK - Returns the ReplicationController object

401: Unauthorized

Examples
