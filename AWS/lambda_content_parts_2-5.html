Step 2: Write Your First Function Code
# lambda_function.py
import json
from datetime import datetime

def lambda_handler(event, context):
    """
    Simple Lambda function for anihpj job portal
    Returns a greeting message with current timestamp
    """
    
    # Log incoming event
    print(f"Received event: {json.dumps(event)}")
    
    # Extract data from event
    name = event.get('name', 'Job Seeker')
    action = event.get('action', 'visited')
    
    # Create response
    response_data = {
        'message': f'Hello {name}! Welcome to anihpj Job Portal',
        'action': action,
        'timestamp': datetime.utcnow().isoformat(),
        'function_name': context.function_name,
        'request_id': context.request_id,
        'platform': 'AWS Lambda Serverless'
    }
    
    # Return response (API Gateway compatible)
    return {
        'statusCode': 200,
        'headers': {
            'Content-Type': 'application/json',
            'Access-Control-Allow-Origin': '*'
        },
        'body': json.dumps(response_data)
    }

📦 Lambda Layers

Layers let you package dependencies separately from your function code, enabling code reuse and smaller deployment packages.

Creating & Using Lambda Layers
# Create a layer for common Python dependencies

# Step 1: Create layer directory structure
mkdir -p lambda-layers/python-deps/python
cd lambda-layers/python-deps

# Step 2: Install dependencies into python/ directory
pip install --target python/ psycopg2-binary boto3 requests Django

# Step 3: Create ZIP file
zip -r python-deps-layer.zip python/

# Step 4: Upload to AWS Lambda
aws lambda publish-layer-version \
    --layer-name anihpj-python-dependencies \
    --description "Common Python deps for anihpj functions" \
    --zip-file fileb://python-deps-layer.zip \
    --compatible-runtimes python3.12 \
    --region us-east-1

# Response includes LayerVersionArn:
# arn:aws:lambda:us-east-1:123456789012:layer:anihpj-python-dependencies:1

# Step 5: Attach layer to Lambda function
aws lambda update-function-configuration \
    --function-name anihpj-job-processor \
    --layers arn:aws:lambda:us-east-1:123456789012:layer:anihpj-python-dependencies:1

import boto3

def lambda_handler(event, context):
    # Read environment variables
    db_host = os.environ['DB_HOST']
    db_name = os.environ['DB_NAME']
    s3_bucket = os.environ['S3_BUCKET_NAME']

    return {'statusCode': 200, 'body': 'Connected successfully'}

🌐 VPC Configuration
Connect Lambda to VPC (for RDS access)
Why VPC Configuration?
• Access RDS databases in private subnets
• Connect to ElastiCache (Redis/Memcached)
• Access resources not exposed to internet

Configuration Steps:

1. Choose VPC
   VPC ID: vpc-12345abc (anihpj-production-vpc)

2. Select Subnets (at least 2 in different AZs)
   • subnet-111 (us-east-1a, private)
   • subnet-222 (us-east-1b, private)

3. Attach Security Group
   Security Group: sg-lambda-rds-access
   • Outbound: All traffic to RDS security group
   • Outbound: HTTPS (443) to internet (for AWS APIs)

4. RDS Security Group (sg-rds-postgres)
   • Inbound: PostgreSQL (5432) from sg-lambda-rds-access

Important Considerations:

⚠️ Cold Start Impact
   VPC Lambda has slower cold starts (~1-2 seconds extra)
   Solution: Use Provisioned Concurrency for critical functions

⚠️ Internet Access
   Lambda in VPC has NO internet access by default
   Solution: Add NAT Gateway or VPC Endpoints

⚠️ IP Addresses
   Lambda consumes ENIs (Elastic Network Interfaces)
   Ensure sufficient IPs in subnets

Terraform Configuration:
```hcl
resource "aws_lambda_function" "anihpj_job_processor" {
  function_name = "anihpj-job-processor"
  runtime       = "python3.12"
  
  vpc_config {
    subnet_ids         = ["subnet-111", "subnet-222"]
    security_group_ids = ["sg-lambda-rds-access"]
  }
  
  environment {
    variables = {
      DB_HOST = aws_rds_cluster.anihpj.endpoint
    }
  }
}
```

Best Practices:
✅ Use VPC only when necessary (RDS, ElastiCache, private resources)
✅ Use multiple subnets across AZs for redundancy
✅ Monitor ENI usage in CloudWatch
✅ Use VPC Endpoints for AWS services (S3, DynamoDB) to avoid NAT costs
✅ Test cold start latency after VPC configuration

💡 Cost Optimization Strategies
🎯 Top 10 Lambda Cost Optimization Tips

1. Right-Size Memory Allocation
   ❌ Bad: 3008 MB when function uses 256 MB
   ✅ Good: Test different memory sizes, use AWS Lambda Power Tuning
   💰 Savings: Up to 80% on compute costs
   
   Tool: AWS Lambda Power Tuning (open-source)
   → Automatically finds optimal memory configuration

2. Reduce Function Duration
   ✅ Optimize code (avoid unnecessary loops)
   ✅ Use connection pooling (database connections)
   ✅ Minimize cold starts (reuse initialization)
   ✅ Parallel processing where possible
   💰 Savings: 30-50% reduction in duration = 30-50% cost savings

3. Use Graviton2 Processors (ARM64)
   ✅ 20% better price/performance than x86
   ✅ Simply change architecture in console
   💰 Savings: 20% cost reduction automatically
   
   Compatibility: Most Python packages support ARM64

4. Minimize Package Size
   ❌ Bad: 100 MB deployment package with unused dependencies
   ✅ Good: Strip to essential packages only, use Layers
   💰 Faster cold starts = Lower costs + Better UX

5. Leverage Free Tier
   ✅ 1M requests/month FREE
   ✅ 400,000 GB-seconds/month FREE
   💰 For small apps: Essentially FREE hosting!

6. Batch Processing
   ❌ Bad: 1 Lambda invocation per message (1000 messages = 1000 invocations)
   ✅ Good: Process 10 messages per invocation (1000 messages = 100 invocations)
   💰 Savings: 90% reduction in request charges

7. Use Asynchronous Invocation
   ✅ For non-critical tasks (emails, logging)
   ✅ No need to wait for response
   💰 Better throughput, potential cost savings from reduced timeouts

8. Reserved Concurrency (Only When Needed)
   ⚠️ Provisioned Concurrency is EXPENSIVE
   ✅ Use only for critical, latency-sensitive functions
   ✅ Scheduled scaling (high concurrency during business hours only)
   💰 Can add 50-100% to costs, use sparingly!

9. Monitor & Set Alarms
   ✅ CloudWatch alarms for cost anomalies
   ✅ AWS Budgets for spending limits
   ✅ Regular cost analysis with Cost Explorer
   💰 Catch runaway costs early

10. Regional Pricing Differences
    ✅ us-east-1 (N. Virginia) usually cheapest
    ✅ Consider data transfer costs if cross-region
    💰 Up to 15% savings by choosing cheaper regions

📊 Interactive Cost Calculator
# Simple Lambda Cost Calculator for anihpj

def calculate_lambda_cost(
    requests_per_month: int,
    memory_mb: int,
    duration_ms: int,
    data_transfer_gb: int = 0
):
    """
    Calculate AWS Lambda costs
    
    Args:
        requests_per_month: Number of function invocations
        memory_mb: Memory allocated (128-10240 MB)
        duration_ms: Average execution time in milliseconds
        data_transfer_gb: Data transferred out (optional)
    
    Returns:
        dict: Breakdown of costs
    """
    # Constants (2025 pricing)
    FREE_REQUESTS = 1_000_000
    FREE_GB_SECONDS = 400_000
    FREE_DATA_TRANSFER_GB = 100
    
    PRICE_PER_MILLION_REQUESTS = 0.20
    PRICE_PER_GB_SECOND = 0.0000166667
    PRICE_PER_GB_DATA_TRANSFER = 0.09
    
    # Calculate request charges
    billable_requests = max(0, requests_per_month - FREE_REQUESTS)
    request_cost = (billable_requests / 1_000_000) * PRICE_PER_MILLION_REQUESTS
    
    # Calculate compute charges
    memory_gb = memory_mb / 1024
    duration_seconds = duration_ms / 1000
    gb_seconds_per_request = memory_gb * duration_seconds
    total_gb_seconds = requests_per_month * gb_seconds_per_request
    
    billable_gb_seconds = max(0, total_gb_seconds - FREE_GB_SECONDS)
    compute_cost = billable_gb_seconds * PRICE_PER_GB_SECOND
    
    # Calculate data transfer charges
    billable_data_transfer = max(0, data_transfer_gb - FREE_DATA_TRANSFER_GB)
    data_transfer_cost = billable_data_transfer * PRICE_PER_GB_DATA_TRANSFER
    
    # Total cost
    total_cost = request_cost + compute_cost + data_transfer_cost
    
    return {
        'request_cost': round(request_cost, 2),
        'compute_cost': round(compute_cost, 2),
        'data_transfer_cost': round(data_transfer_cost, 2),
        'total_cost': round(total_cost, 2),
        'cost_per_request': round(total_cost / requests_per_month, 6) if requests_per_month > 0 else 0,
        'total_gb_seconds': round(total_gb_seconds, 2),
        'free_tier_used': requests_per_month <= FREE_REQUESTS and total_gb_seconds <= FREE_GB_SECONDS
    }

    duration_ms=200,                  # 200ms avg

)
print("Small Deployment (Dev/Staging):")
