🚀 Getting Started with Systems Manager

Follow these steps to start managing your anihpj Django servers with Systems Manager:

📋 Step 1: Prerequisites
  • EC2 instances running (anihpj-web-1, web-2, web-3)
  • Instances have internet connectivity or VPC endpoints
  • Ubuntu 22.04/Amazon Linux (SSM Agent pre-installed)
  • IAM permissions to use Systems Manager
🔐 Step 2: Create IAM Role
  • Create IAM role: anihpj-SSM-Role
  • Attach policy: AmazonSSMManagedInstanceCore
  • Attach role to all anihpj EC2 instances
  • Instances will appear in Fleet Manager
✅ Step 3: Verify Connection
  • Open Systems Manager console
  • Go to Fleet Manager
  • Verify all 3 instances show "Online" status
  • Start using Session Manager, Run Command
🔧 Step 4: Configure Features
  • Set up Parameter Store for secrets
  • Configure Patch Manager baselines
  • Enable Session Manager logging
  • Create maintenance windows
📋 Create the IAM Role for Systems Manager
# Create IAM role for SSM
aws iam create-role \
    --role-name anihpj-SSM-Role \
    --assume-role-policy-document '{
        "Version": "2012-10-17",
        "Statement": [{
            "Effect": "Allow",
            "Principal": {"Service": "ec2.amazonaws.com"},
            "Action": "sts:AssumeRole"
        }]
    }'

# Attach the SSM managed policy
aws iam attach-role-policy \
    --role-name anihpj-SSM-Role \
    --policy-arn arn:aws:iam::aws:policy/AmazonSSMManagedInstanceCore

# Create instance profile
aws iam create-instance-profile \
    --instance-profile-name anihpj-SSM-InstanceProfile

# Add role to instance profile
aws iam add-role-to-instance-profile \
    --instance-profile-name anihpj-SSM-InstanceProfile \
    --role-name anihpj-SSM-Role

# Attach the instance profile to your EC2 instances
aws ec2 associate-iam-instance-profile \
    --instance-id i-1234567890abcdef0 \
    --iam-instance-profile Name=anihpj-SSM-InstanceProfile

# ✅ Repeat for all anihpj Django web servers (web-1, web-2, web-3)
⏱️ Wait 5-10 minutes: After attaching the IAM role, it takes a few minutes for the SSM Agent to register with Systems Manager. Check the Fleet Manager console to see when instances appear as "Online".
