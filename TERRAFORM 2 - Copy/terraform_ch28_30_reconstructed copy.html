aws_lambda_function

Official Docs: aws_lambda_function | Terraform Registry: hashicorp/aws | AWS Docs: AWS Lambda Developer Guide

Provides an AWS Lambda Function resource to deploy and manage serverless functions for your anihpj Django application's API handlers, background workers, and event processors.

Full Description

The aws_lambda_function resource creates and manages AWS Lambda functions, enabling serverless compute for your anihpj Django project. Lambda functions can serve as API backends, process events from S3, SQS, DynamoDB Streams, and other AWS services, or run scheduled tasks - all without provisioning or managing servers.

For the anihpj Django project with its Skills, Author, Location, User, and JobPost models, Lambda functions are ideal for:

  • API Handlers: Process REST/GraphQL requests for JobPost CRUD operations
  • Background Workers: Handle Skills extraction, Location geocoding, Author validation
  • Event Processors: React to S3 uploads (resumes), DynamoDB changes (audit logs)
  • Scheduled Tasks: Clean up expired JobPosts, send notification digests to Authors
  • Data Pipelines: Transform and load job posting data for analytics

Lambda automatically scales from zero to thousands of concurrent executions, charges only for compute time consumed (minimum 1ms billing), and integrates natively with AWS services. For CPU-intensive Django operations like Skills matching algorithms or Location distance calculations, configure appropriate memory (CPU scales proportionally) and timeout settings.

Syntax

# Basic Lambda function from ZIP file
resource "aws_lambda_function" "basic" {
  function_name = "anihpj-api-handler"
  description   = "Handles API requests for anihpj JobPost operations"
  
  # Deployment package (ZIP file)
  filename         = "lambda.zip"
  source_code_hash = filebase64sha256("lambda.zip")
  
  # Runtime configuration
  runtime = "python3.12"
  handler = "handler.main"
  
  # Execution role (required)
  role = aws_iam_role.lambda_exec.arn
  
  # Resource allocation
  memory_size = 256       # MB (128-10240)
  timeout     = 30        # seconds (1-900)
  
  # Environment variables
  environment {
    variables = {
      DJANGO_SETTINGS_MODULE = "anihpj.settings"
      DATABASE_URL           = var.database_url
    }
  }
}

# Lambda function from S3 bucket
resource "aws_lambda_function" "from_s3" {
  function_name = "anihpj-worker"
  
  s3_bucket         = aws_s3_bucket.deployments.id
  s3_key            = "lambda/worker-v1.0.0.zip"
  s3_object_version = "abc123"  # Optional: specific version
  
  runtime = "python3.12"
  handler = "worker.process"
  role    = aws_iam_role.lambda_exec.arn
}

# Lambda function from container image
resource "aws_lambda_function" "container" {
  function_name = "anihpj-container"
  package_type  = "Image"
  image_uri     = "${aws_ecr_repository.lambda.repository_url}:latest"
  role          = aws_iam_role.lambda_exec.arn
  
  image_config {
    command           = ["handler.main"]
    entry_point       = ["/lambda-entrypoint.sh"]
    working_directory = "/app"
  }
}

# Lambda function with VPC access
resource "aws_lambda_function" "vpc_enabled" {
  function_name = "anihpj-rds-accessor"
  
  vpc_config {
    subnet_ids         = var.private_subnet_ids
    security_group_ids = [aws_security_group.lambda.id]
  }
  
  # ... other configuration
}

Parameters Description

Parameter Description
function_name Required. Unique name for the Lambda function (1-64 characters). For anihpj, use prefixes like anihpj-api-, anihpj-worker- to organize functions by purpose.
role Required. ARN of the IAM role that Lambda assumes when executing. Must have lambda.amazonaws.com as trusted entity and appropriate permissions for DynamoDB, S3, etc.
runtime Required for ZIP. Runtime identifier. For Django: python3.12, python3.11. Node.js: nodejs20.x. Not needed for container images.
handler Required for ZIP. Entry point in format filename.function_name. Example: jobpost_handler.process_request for Django view handlers.
filename Path to local ZIP deployment package. Use with source_code_hash for proper update detection when code changes.
source_code_hash Base64-encoded SHA256 hash of the package. Use filebase64sha256("lambda.zip") to detect code changes and trigger updates.
s3_bucket S3 bucket containing deployment package. Used for larger packages (>50MB direct upload limit) or CI/CD pipelines.
s3_key S3 object key for the deployment package. Example: deployments/anihpj-api/v1.2.0.zip.
image_uri ECR image URI for container-based Lambda. Format: account.dkr.ecr.region.amazonaws.com/repo:tag.
memory_size Memory allocation in MB (128-10240). CPU scales proportionally. Default: 128. For Django with ORM: recommend 256-512MB minimum.
timeout Maximum execution time in seconds (1-900). Default: 3. For API handlers: 10-30s. For batch processing: 300-900s.
environment Block containing variables map. Pass Django settings, database URLs, API keys. Total size limit: 4KB.
vpc_config Block with subnet_ids and security_group_ids for VPC access. Required to connect to RDS, ElastiCache in private subnets.
reserved_concurrent_executions Maximum concurrent executions (-1 = unreserved, 0 = disabled). Use to limit database connections or manage costs.
publish Whether to publish a new version on each update. Default: false. Enable for aliases and gradual deployments.
architectures List with architecture: ["x86_64"] or ["arm64"]. ARM (Graviton2) offers ~20% better price/performance.
ephemeral_storage Block with size in MB (512-10240) for /tmp storage. Increase for large file processing operations.
layers List of Lambda Layer ARNs (max 5). Use for shared Django dependencies, utilities, or SDK packages.
dead_letter_config Block with target_arn (SQS/SNS) for failed async invocations. Essential for JobPost processing reliability.
tracing_config Block with mode (Active or PassThrough) for X-Ray tracing integration.

Attributes

Attribute Description
arn The ARN identifying the Lambda function. Format: arn:aws:lambda:region:account:function:name.
invoke_arn ARN used for API Gateway integration. Format uses apigateway service principal.
qualified_arn ARN with version suffix when publish = true. Use for immutable references.
qualified_invoke_arn Invoke ARN with version for API Gateway to specific version.
version Published version number ($LATEST if unpublished). Increments on each publish.
last_modified ISO 8601 timestamp of last function update. Useful for deployment tracking.
code_sha256 Base64-encoded SHA256 hash of deployed code. Verify deployment integrity.
signing_job_arn ARN of the signing job (if code signing enabled). For compliance requirements.
signing_profile_version_arn ARN of the signing profile version used.

Methods

Deploy from Archive File

Syntax: data "archive_file" with filename + source_code_hash

Parameters: source_dir (string) - Directory containing Lambda code; output_path (string) - Path for generated ZIP

For anihpj Django handlers, package your Python code with dependencies into a ZIP. Use archive_file data source to automatically create the package during terraform plan.

Deploy from S3

Syntax: s3_bucket + s3_key + s3_object_version

Parameters: s3_bucket (string) - Bucket name; s3_key (string) - Object path; s3_object_version (string) - Version ID

For CI/CD pipelines, upload packages to S3 and reference them. Supports versioning for rollbacks.

Deploy Container Image

Syntax: package_type = "Image" + image_uri

Parameters: image_uri (string) - ECR URI with tag or digest

For complex Django applications with many dependencies, use container images (up to 10GB vs 250MB ZIP limit).

Update Environment Variables

Syntax: environment { variables = { KEY = "value" } }

Parameters: variables (map) - Key-value pairs for runtime environment

Pass Django settings, database credentials (from Secrets Manager), feature flags to your Lambda functions.

Return Value

On successful creation, Terraform returns the Lambda function's ARN, invoke ARN (for API Gateway), version, and deployment metadata. These outputs are critical for integrating with API Gateway, EventBridge rules, and other AWS services that invoke the function.

Return Value Description
Success Function created/updated successfully. Terraform state reflects new configuration.
InvalidParameterValueException Invalid parameter (runtime, memory, handler format). Check AWS documentation for valid values.
ResourceConflictException Function already exists with different configuration. Import existing resource or use unique name.
CodeStorageExceededException Account storage limit exceeded (75GB default). Delete unused function versions.
InvalidCodeSignatureException Code signature validation failed. Verify signing configuration.

Create a Lambda function that handles REST API requests for JobPost CRUD operations in the anihpj Django application, storing data in DynamoDB.

# Create deployment package from source directory
data "archive_file" "jobpost_api" {
  type        = "zip"
  source_dir  = "${path.module}/../lambda/jobpost_api"
  output_path = "${path.module}/builds/jobpost_api.zip"
}

# Lambda function for JobPost API
resource "aws_lambda_function" "jobpost_api" {
  function_name = "anihpj-jobpost-api"
  description   = "REST API handler for JobPost CRUD operations"
  
  # Deployment package
  filename         = data.archive_file.jobpost_api.output_path
  source_code_hash = data.archive_file.jobpost_api.output_base64sha256
  
  # Runtime configuration
  runtime       = "python3.12"
  handler       = "handler.main"
  architectures = ["arm64"]  # 20% better price/performance
  
  # Execution role
  role = aws_iam_role.lambda_jobpost.arn
  
  # Resource allocation for Django ORM operations
  memory_size = 512   # 512MB for comfortable Django operation
  timeout     = 30    # 30 seconds for API requests
  
  # Ephemeral storage for temporary files
  ephemeral_storage {
    size = 1024  # 1GB /tmp
  }
  
  # Environment variables
  environment {
    variables = {
      ENVIRONMENT          = var.environment
      DYNAMODB_TABLE       = aws_dynamodb_table.jobposts.name
      LOG_LEVEL            = "INFO"
      SKILLS_TABLE         = aws_dynamodb_table.skills.name
      AUTHOR_TABLE         = aws_dynamodb_table.authors.name
      LOCATION_TABLE       = aws_dynamodb_table.locations.name
    }
  }
  
  # X-Ray tracing for performance monitoring
  tracing_config {
    mode = "Active"
  }
  
  # Dead letter queue for failed invocations
  dead_letter_config {
    target_arn = aws_sqs_queue.jobpost_dlq.arn
  }
  
  # Limit concurrent executions to protect DynamoDB
  reserved_concurrent_executions = 100
  
  # Publish version for alias-based deployments
  publish = true
  
  tags = {
    Project     = "anihpj"
    Component   = "jobpost-api"
    Environment = var.environment
    ManagedBy   = "terraform"
  }
}

Example 2: Skills Processing Worker with SQS Trigger

Create a Lambda worker function that processes Skills extraction requests from an SQS queue, analyzing JobPost descriptions and storing extracted skills in DynamoDB.

# SQS Queue for Skills processing requests
resource "aws_sqs_queue" "skills_processing" {
  name                       = "anihpj-skills-processing"
  visibility_timeout_seconds = 300  # 5 min (6x Lambda timeout)
  message_retention_seconds  = 1209600  # 14 days
  delay_seconds             = 0
  receive_wait_time_seconds = 20  # Long polling
  
  # Dead letter queue
  redrive_policy = jsonencode({
    deadLetterTargetArn = aws_sqs_queue.skills_dlq.arn
    maxReceiveCount     = 3
  })
}

resource "aws_sqs_queue" "skills_dlq" {
  name                      = "anihpj-skills-dlq"
  message_retention_seconds = 1209600
}

# Lambda function for Skills extraction
resource "aws_lambda_function" "skills_worker" {
  function_name = "anihpj-skills-worker"
  description   = "Extracts and processes Skills from JobPost descriptions"
  
  filename         = data.archive_file.skills_worker.output_path
  source_code_hash = data.archive_file.skills_worker.output_base64sha256
  
  runtime       = "python3.12"

  architectures = ["arm64"]
  
  role = aws_iam_role.skills_worker.arn

    variables = {
      SKILLS_TABLE       = aws_dynamodb_table.skills.name

      PROCESSING_QUEUE   = aws_sqs_queue.skills_processing.url
      BATCH_SIZE         = "10"
      CONFIDENCE_THRESHOLD = "0.75"
    }
  }
  
  # Reserved concurrency to limit parallel processing
  reserved_concurrent_executions = 10
  
  tags = {
    Project   = "anihpj"

  }

Example 3: Author Notification Service with SNS

Create a Lambda function that sends notifications to Authors when new JobPosts matching their skills are created, using SNS for multi-channel delivery.

# SNS Topic for Author notifications

      NOTIFICATION_TYPE = "JOB_MATCH"

    Project   = "anihpj"

aws_lambda_layer_version

Official Docs: aws_lambda_layer_version | Terraform Registry: hashicorp/aws | AWS Docs: Lambda Layers

Provides an AWS Lambda Layer resource to package and share common dependencies, libraries, and utilities across multiple Lambda functions in your anihpj Django application.

Full Description

The aws_lambda_layer_version resource creates Lambda Layers - reusable packages that contain libraries, custom runtimes, or other dependencies. For the anihpj Django project, layers are essential for sharing common code across multiple Lambda functions handling Skills, Author, Location, User, and JobPost operations.

Key benefits of Lambda Layers for anihpj:

  • Shared Dependencies: Package Django ORM, Boto3, Requests once for all functions
  • Smaller Deployments: Function code stays small, layers handle heavy dependencies
  • Version Control: Update shared libraries independently of function code
  • Code Reuse: Share anihpj utility functions (serializers, validators) across functions
  • Cost Reduction: Reduce deployment package size and build times

Lambda supports up to 5 layers per function with a combined unzipped size limit of 250MB. Layers are mounted at /opt in the Lambda execution environment, with Python packages available at /opt/python.

Syntax

# Basic Lambda Layer from local ZIP
resource "aws_lambda_layer_version" "dependencies" {
  layer_name          = "anihpj-dependencies"
  description         = "Common Python dependencies for anihpj"
  
  # Deployment package
  filename            = "layers/dependencies.zip"
  source_code_hash    = filebase64sha256("layers/dependencies.zip")
  
  # Compatible runtimes
  compatible_runtimes = ["python3.11", "python3.12"]
  
  # Compatible architectures
  compatible_architectures = ["x86_64", "arm64"]
  
  # License information
  license_info = "MIT"
}

# Lambda Layer from S3
resource "aws_lambda_layer_version" "from_s3" {
  layer_name = "anihpj-utils"
  
  s3_bucket         = aws_s3_bucket.layers.id
  s3_key            = "layers/utils-v1.0.0.zip"
  s3_object_version = "abc123"
  
  compatible_runtimes = ["python3.12"]
}

# Reference layer in Lambda function
resource "aws_lambda_function" "with_layers" {
  function_name = "anihpj-api"
  # ... other config
  
  layers = [
    aws_lambda_layer_version.dependencies.arn,
    aws_lambda_layer_version.utils.arn,
    "arn:aws:lambda:us-east-1:123456789012:layer:aws-sdk-pandas:1"  # Public layer
  ]
}

Parameters Description

Parameter Description
layer_name Required. Unique name for the layer (1-140 characters). Use prefixes like anihpj-deps-, anihpj-utils- to organize layers by purpose.
filename Path to local ZIP file containing layer content. Layer ZIP structure must have libraries in correct paths (python/ for Python packages).
source_code_hash Base64-encoded SHA256 hash for detecting changes. Use filebase64sha256() function.
s3_bucket S3 bucket containing the layer ZIP. Use for large layers or CI/CD pipelines.
s3_key S3 object key for the layer ZIP file.
compatible_runtimes List of compatible Lambda runtimes. For Django: ["python3.11", "python3.12"].
compatible_architectures List of compatible architectures: ["x86_64"], ["arm64"], or both.
description Description of the layer's purpose and contents. Document included packages and versions.
license_info Layer license information (max 512 characters). Include licenses of bundled packages.
skip_destroy When true, layer versions are retained when updating. Default: false. Enable for rollback capability.

Attributes

Attribute Description
arn ARN of the Lambda Layer with version. Format: arn:aws:lambda:region:account:layer:name:version.
layer_arn ARN without version suffix. Use to reference latest version.
version Version number of this layer. Increments with each new deployment.
created_date ISO 8601 timestamp when layer version was created.
code_sha256 Base64-encoded SHA256 hash of deployed layer content.
signing_job_arn ARN of the signing job if code signing is enabled.
signing_profile_version_arn ARN of the signing profile version used.

Methods

Create Layer Package

Syntax: pip install -r requirements.txt -t python/

Parameters: requirements.txt (file) - List of Python packages; python/ (directory) - Target directory following Lambda layer structure

Package Python dependencies into the correct directory structure for Lambda layers. Must use python/ or python/lib/python3.x/site-packages/ path.

Share Layer Across Accounts

Syntax: aws_lambda_layer_version_permission

Parameters: principal (string) - AWS account ID or "*" for public; action (string) - "lambda:GetLayerVersion"

Grant other AWS accounts permission to use your layer. Use for shared organization layers.

Return Value

Returns the layer ARN with version number, which is used in Lambda function configurations. Each create/update produces a new version number; old versions remain accessible until explicitly deleted.

Return Value Description

aws_lambda_permission

Official Docs: aws_lambda_permission | Terraform Registry: hashicorp/aws | AWS Docs: Resource-based Policies

Provides an AWS Lambda Permission resource to grant other AWS services and accounts permission to invoke your Lambda functions in the anihpj Django application.

Full Description

The aws_lambda_permission resource manages resource-based policies for Lambda functions, controlling which AWS services, accounts, and organizations can invoke the function. For the anihpj Django project, permissions are essential for:

  • API Gateway: Allow REST/HTTP APIs to invoke JobPost, Skills, Author handlers
  • S3: Allow bucket events to trigger resume processing for User uploads
  • SNS/SQS: Allow messaging services to invoke notification handlers
  • EventBridge: Allow scheduled rules to trigger batch JobPost processing
  • CloudWatch: Allow alarms to invoke error handling functions
  • Cross-Account: Allow partner accounts to invoke shared anihpj APIs

Without proper permissions, AWS services cannot invoke your Lambda functions even if the IAM role has the correct execution permissions. Lambda permissions are resource-based (attached to the function) rather than identity-based (attached to the caller).

Syntax

# Basic permission for API Gateway
resource "aws_lambda_permission" "api_gateway" {
  statement_id  = "AllowAPIGatewayInvoke"
  action        = "lambda:InvokeFunction"
  function_name = aws_lambda_function.api.function_name
  principal     = "apigateway.amazonaws.com"
  source_arn    = "${aws_api_gateway_rest_api.main.execution_arn}/*/*"
}

# Permission for S3 bucket events
resource "aws_lambda_permission" "s3_trigger" {
  statement_id  = "AllowS3Invoke"
  action        = "lambda:InvokeFunction"
  function_name = aws_lambda_function.processor.function_name
  principal     = "s3.amazonaws.com"
  source_arn    = aws_s3_bucket.uploads.arn
  source_account = data.aws_caller_identity.current.account_id
}

# Permission for SNS topic
resource "aws_lambda_permission" "sns_invoke" {
  statement_id  = "AllowSNSInvoke"
  action        = "lambda:InvokeFunction"
  function_name = aws_lambda_function.notifier.function_name
  principal     = "sns.amazonaws.com"
  source_arn    = aws_sns_topic.notifications.arn
}

# Permission for EventBridge rule
resource "aws_lambda_permission" "eventbridge" {
  statement_id  = "AllowEventBridgeInvoke"
  action        = "lambda:InvokeFunction"
  function_name = aws_lambda_function.scheduler.function_name
  principal     = "events.amazonaws.com"
  source_arn    = aws_cloudwatch_event_rule.daily.arn
}

# Cross-account permission
resource "aws_lambda_permission" "cross_account" {
  statement_id   = "AllowCrossAccountInvoke"
  action         = "lambda:InvokeFunction"
  function_name  = aws_lambda_function.shared_api.function_name
  principal      = "arn:aws:iam::987654321098:root"
  source_account = "987654321098"
}

Parameters Description

Parameter Description
statement_id Required. Unique identifier for the permission statement (1-100 chars). Use descriptive names like AllowAPIGatewayInvoke, AllowS3JobPostUpload.
action Required. Lambda action to allow. Usually lambda:InvokeFunction. Can also be lambda:InvokeFunctionUrl for function URLs.
function_name Required. Name, ARN, or partial ARN of the Lambda function. Reference aws_lambda_function.name.function_name for clarity.
principal Required. AWS service principal (e.g., apigateway.amazonaws.com, s3.amazonaws.com) or account ARN.
source_arn ARN of the source resource that will invoke. Use with wildcards for API Gateway: ${api.execution_arn}/*/*.
source_account AWS account ID of the source. Required with source_arn for S3 to prevent confused deputy attacks.
qualifier Lambda alias or version to grant permission to. Use $LATEST or alias name for specific targeting.
principal_org_id AWS Organization ID for org-wide access. Use with principal = "*" for org-scoped permissions.
function_url_auth_type Auth type for function URL invocations: AWS_IAM or NONE.
event_source_token Token for Alexa Smart Home functions. Not typically used for Django applications.

Attributes

Attribute Description
id Permission ID in format function_name/statement_id.
statement_id The statement ID provided in configuration.

Methods

Grant API Gateway Permission

Syntax: principal = "apigateway.amazonaws.com" + source_arn = "${api.execution_arn}/*/*"

Parameters: source_arn (string) - API execution ARN with method/path wildcards

Allow API Gateway to invoke Lambda. Use /*/* for all methods and paths, or specify like /*/POST/jobposts.

Grant S3 Permission

Syntax: principal = "s3.amazonaws.com" + source_arn + source_account

Parameters: source_arn (string) - S3 bucket ARN; source_account (string) - Account ID for security

Allow S3 to invoke Lambda on object events. Always include source_account for security.

Grant Cross-Account Permission

Syntax: principal = "arn:aws:iam::ACCOUNT:root"

Parameters: principal (string) - IAM ARN of trusted account

Allow other AWS accounts to invoke your Lambda function for shared services.

Return Value

Creates a resource-based policy statement on the Lambda function. The permission is immediately effective upon creation. Returns the statement ID for reference.

Return Value Description
Success Permission added to function's resource-based policy.
ResourceConflictException Permission statement with same ID already exists.
PolicyLengthExceededException Resource policy exceeds 20KB limit. Consolidate statements.

⚠️ Lambda Permission Security Best Practices:
  • Source ARN: Always specify source_arn to restrict which resources can invoke
  • Source Account: For S3 triggers, always include source_account to prevent confused deputy attacks
  • Wildcards: Avoid * in principals unless using principal_org_id
  • Statement IDs: Use descriptive, unique statement IDs for easy auditing
  • Policy Size: Resource policy limit is 20KB; consolidate permissions if needed
