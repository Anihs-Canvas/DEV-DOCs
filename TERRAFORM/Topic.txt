### C. Glossary
- Terraform Terminology
  - Resource: Infrastructure object managed by Terraform
  - Data Source: Read-only information about existing resources
  - Provider: Plugin that interacts with cloud/service APIs
  - Module: Reusable collection of Terraform configurations
  - State: Record of managed infrastructure
  - Backend: Remote storage for state files
  - Workspace: Named instances of state within backend
  - Plan: Preview of changes Terraform will make
  - Apply: Execute planned infrastructure changes
  - Provisioner: Script execution during resource lifecycle
  - Dynamic Block: Programmatically generate nested blocks
  - Meta-Argument: Special arguments affecting resource behavior
  - Sentinel: Policy as code for Terraform Cloud/Enterprise
  - HCL: HashiCorp Configuration Language
  - Lock File: Provider version constraints record
- AWS Terminology
  - VPC: Virtual Private Cloud
  - Subnet: Network segment within VPC
  - Security Group: Virtual firewall for instances
  - NACL: Network Access Control List
  - IGW: Internet Gateway
  - NAT Gateway: Network Address Translation service
  - ALB: Application Load Balancer
  - ASG: Auto Scaling Group
  - AMI: Amazon Machine Image
  - EBS: Elastic Block Store
  - IAM: Identity and Access Management
  - ARN: Amazon Resource Name
  - AZ: Availability Zone
  - Region: Geographic AWS data center location
  - CloudFormation: AWS native IaC service
- IaC Terminology
  - Infrastructure as Code: Managing infrastructure through code
  - Declarative: Describe desired state, not steps
  - Imperative: Define explicit steps to execute
  - Idempotent: Same operation produces same result
  - Drift: Difference between actual and desired state
  - Orchestration: Coordinating multiple infrastructure components
  - Provisioning: Creating and configuring resources
  - GitOps: Using Git as source of truth for infrastructure
  - Continuous Deployment: Automated deployment pipeline
  - Immutable Infrastructure: Replace rather than update

### F. Troubleshooting Decision Tree
```
Problem: terraform init fails
├─ Provider download issue?
│  ├─ Check internet connection
│  ├─ Verify provider source
│  └─ Check version constraints
├─ Backend configuration issue?
│  ├─ Verify backend credentials
│  ├─ Check bucket/table existence
│  └─ Review backend configuration
└─ Lock file conflict?
   ├─ Delete .terraform.lock.hcl
   ├─ Run terraform init -upgrade
   └─ Commit new lock file

Problem: terraform plan shows unexpected changes
├─ State drift?
│  ├─ Run terraform refresh
│  ├─ Check manual changes
│  └─ Review CloudTrail logs
├─ Provider version change?
│  ├─ Check .terraform.lock.hcl
│  ├─ Review provider changelog
│  └─ Pin provider version
└─ Configuration change?
   ├─ Check git diff
   ├─ Review module updates
   └─ Verify variable values

Problem: terraform apply fails
├─ Authentication issue?
│  ├─ Check AWS credentials
│  ├─ Verify IAM permissions
│  └─ Test with AWS CLI
├─ Resource limit exceeded?
│  ├─ Check service quotas
│  ├─ Request limit increase
│  └─ Use different region
├─ Dependency issue?
│  ├─ Review error message
│  ├─ Check depends_on
│  └─ Verify resource order
└─ Timeout issue?
   ├─ Increase timeout
   ├─ Check resource state
   └─ Review provider logs

Problem: State lock errors
├─ Previous apply crashed?
│  ├─ Check for running processes
│  ├─ Verify lock table
│  └─ Force unlock (careful!)
├─ Concurrent applies?
│  ├─ Coordinate team members
│  ├─ Use proper CI/CD workflow
│  └─ Implement approval gates
└─ Backend issue?
   ├─ Check backend connectivity
   ├─ Verify backend permissions
   └─ Review backend logs
```

## 🚀 MIGRATION GUIDES

### CloudFormation to Terraform
1. **Assess Current Stack**
   - Export CloudFormation template
   - Document resource dependencies
   - Identify drift from templates
2. **Import Strategy**
   - Use terraform import for resources
   - Generate HCL from AWS Console
   - Tools: cf-to-tf, former2
3. **Gradual Migration**
   - Migrate non-critical resources first
   - Test thoroughly in dev/staging
   - Parallel run for validation
4. **Cut-Over Plan**
   - Blue/green approach
   - DNS failover preparation
   - Rollback procedures

### Manual Infrastructure to Terraform
1. **Discovery Phase**
   - Document existing resources
   - Create dependency maps
   - Identify shared resources
2. **Import Process**
   - terraform import commands
   - State file organization
   - Tag standardization
3. **Refactoring**
   - Extract modules
   - Standardize naming
   - Apply consistent tagging
4. **Validation**
   - terraform plan -refresh-only
   - Compare with AWS Console
   - Test in isolated environment

## 🎯 SUCCESS METRICS & KPIs

### Infrastructure as Code Maturity
- **Level 1 - Ad Hoc**: Manual changes, no version control
- **Level 2 - Managed**: Version controlled, manual apply
- **Level 3 - Defined**: CI/CD integration, code review
- **Level 4 - Measured**: Automated testing, drift detection
- **Level 5 - Optimized**: Policy as code, auto-remediation

### Key Performance Indicators
- Infrastructure Provisioning Time: Target < 30 minutes
- Deployment Frequency: Target > 10/day
- Change Failure Rate: Target < 5%
- Mean Time to Recovery: Target < 1 hour
- Infrastructure Drift: Target < 1%
- Cost Variance: Target ±5% of budget
- Security Compliance: Target 100%
