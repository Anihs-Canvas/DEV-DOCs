1.1 Introduction to Terraform

🔷 What is Terraform?

Terraform Overview

Terraform is an open-source infrastructure as code (IaC) tool created by HashiCorp that allows you to define, provision, and manage cloud infrastructure using a declarative configuration language.

📝
Declarative Infrastructure Management

Define your desired infrastructure state using HashiCorp Configuration Language (HCL). Terraform figures out how to achieve that state.

resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"
  
  tags = {
    Name = "MyWebServer"
  }
}

🏗️ Infrastructure as Code (IaC) Concept

❌ Traditional Infrastructure Management
  • 🖱️ Manual provisioning through web consoles
  • 📋 Documentation in wikis or documents
  • 🔄 Inconsistent environments
  • ❌ Hard to reproduce
  • ⏰ Time-consuming deployments
  • 🐛 Prone to human error
  • 📝 No version control
Typical Workflow:
  1. Log into AWS Console
  2. Click through EC2 wizard
  3. Manually configure security groups
  4. Document settings in spreadsheet
  5. Repeat for each environment 😰
VS
✅ Infrastructure as Code (Terraform)
  • ⚡ Automated provisioning through code
  • 📄 Infrastructure defined in code files
  • 🎯 Consistent and reproducible
  • ✅ Version controlled with Git
  • 🚀 Fast, repeatable deployments
  • 🔍 Code review and testing
  • 📊 Complete audit trail
Terraform Workflow:
  1. Write configuration in HCL
  2. Run terraform plan
  3. Review changes
  4. Apply with terraform apply
  5. Deploy to any environment instantly 🎉
🔄
Version Control

🔁
Reusability

🧪
Testing

Test infrastructure changes before applying them. Automated validation and compliance checks.

🤝
Collaboration

Team-based workflows with code reviews, pull requests, and shared state management.

⚖️ Terraform vs Other IaC Tools

Tool Approach Language Multi-Cloud State Management Best For
🟣 Terraform Declarative HCL / JSON ✅ Yes ✅ Built-in Multi-cloud infrastructure provisioning
🔷 CloudFormation Declarative YAML / JSON ❌ AWS Only ✅ Built-in AWS-native infrastructure
📘 ARM Templates Declarative JSON ❌ Azure Only ✅ Built-in Azure-native infrastructure
🎭 Ansible Procedural YAML ⚠️ Limited ❌ No Configuration management, provisioning
🐍 Pulumi Declarative Python, TypeScript, Go ✅ Yes ✅ Built-in Developers preferring general-purpose languages
🔶 CDK (AWS/Terraform) Procedural TypeScript, Python, Java ⚠️ Varies ✅ Via Backend Programmatic infrastructure definition
✅ Choose Terraform When:
  • 🌐 You need multi-cloud or hybrid cloud infrastructure
  • 🔄 You want infrastructure versioning and rollback
  • 🏢 Working with multiple providers (AWS, Datadog, PagerDuty, etc.)
  • 👥 Team collaboration with state locking is essential
  • 📦 You want to use community modules and providers
  • 🎯 Declarative approach fits your workflow
🤔 Consider Alternatives When:
  • ☁️ Single cloud provider only (CloudFormation for AWS-only)
  • 🔧 Configuration management is primary need (Ansible)
  • 💻 Team prefers general-purpose programming languages (Pulumi)
  • 🔄 Procedural/imperative approach preferred (Scripts, Ansible)
  • 🎛️ Complex application logic needed (CDK)

📝 1. Configuration Files

Written in HashiCorp Configuration Language (HCL) with .tf extension

  • main.tf - Primary configuration
  • variables.tf - Input variables
  • outputs.tf - Output values
  • terraform.tfvars - Variable values
🔄 2. Terraform Init

Initialize working directory and download providers

$ terraform init
  • ✅ Download required providers
  • ✅ Initialize backend for state storage
  • ✅ Download modules
  • ✅ Create .terraform directory
📊 3. State Management

Terraform maintains infrastructure state in terraform.tfstate

Local State: Stored on local filesystem
Remote State: Stored in S3, Azure Blob, Terraform Cloud (✨ Recommended for teams)
🔌 4. Provider Plugins

Providers implement resource types and communicate with APIs

terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

🚀 Getting Started with Terraform

Step 1
📥 Install Terraform
🪟 Windows
# Using Chocolatey
choco install terraform

# Or download from https://www.terraform.io/downloads
🍎 macOS
# Using Homebrew
brew tap hashicorp/tap
brew install hashicorp/tap/terraform
🐧 Linux
# Ubuntu/Debian
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
sudo apt-get update && sudo apt-get install terraform
Verify Installation: 
$ terraform version
Terraform v1.6.0
Step 2
☁️ Set Up Cloud Provider Credentials
🟠 AWS Setup: 
# Install AWS CLI
aws configure

# Or set environment variables
export AWS_ACCESS_KEY_ID="your_access_key"
export AWS_SECRET_ACCESS_KEY="your_secret_key"
export AWS_DEFAULT_REGION="us-east-1"
🔵 Azure Setup: 
# Install Azure CLI and login
az login

# Terraform will use Azure CLI credentials automatically
🔴 GCP Setup: 
# Install gcloud CLI and authenticate
gcloud auth application-default login

# Or use service account key
export GOOGLE_APPLICATION_CREDENTIALS="path/to/key.json"
Step 3
📝 Create Your First Configuration

Create a file named main.tf with your infrastructure definition:

main.tf
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  tags = {
    Name = "MyFirstTerraformInstance"
  }
}
Step 4
▶️ Run Terraform Commands
1️⃣ Initialize
$ terraform init

Downloads providers and prepares working directory

2️⃣ Validate
$ terraform validate

Checks configuration syntax

3️⃣ Plan
$ terraform plan

Preview changes without applying

4️⃣ Apply
$ terraform apply

Create the infrastructure

5️⃣ Destroy (optional)
$ terraform destroy

Remove all resources
