1. Introduction

Understanding Bash and the shell environment

This text is a brief description of the features that are present in the Bash shell (version 5.3, 18 May 2025). The Bash home page is http://www.gnu.org/software/bash/.

This is Edition 5.3, last updated 18 May 2025, of The GNU Bash Reference Manual, for Bash, Version 5.3.

Bash contains features that appear in other popular shells, and some features that only appear in Bash. Some of the shells that Bash has borrowed concepts from are the Bourne Shell (sh), the Korn Shell (ksh), and the C-shell (csh and its successor, tcsh). The following menu breaks the features up into categories, noting which features were inspired by other shells and which are specific to Bash.

Note: This manual is meant as a brief introduction to features found in Bash. The Bash manual page should be used as the definitive reference on shell behavior.

1.1 What is Bash?

Bash is the shell, or command language interpreter, for the GNU operating system. The name is an acronym for the 'Bourne-Again SHell', a pun on Stephen Bourne, the author of the direct ancestor of the current Unix shell sh, which appeared in the Seventh Edition Bell Labs Research version of Unix.

Bash is largely compatible with sh and incorporates useful features from the Korn shell ksh and the C shell csh. It is intended to be a conformant implementation of the IEEE POSIX Shell and Tools portion of the IEEE POSIX specification (IEEE Standard 1003.1). It offers functional improvements over sh for both interactive and programming use.

While the GNU operating system provides other shells, including a version of csh, Bash is the default shell. Like other GNU software, Bash is quite portable. It currently runs on nearly every version of Unix and a few other operating systems − independently-supported ports exist for Windows and other platforms.

1.2 What is a shell?

At its base, a shell is simply a macro processor that executes commands. The term macro processor means functionality where text and symbols are expanded to create larger expressions.

A Unix shell is both a command interpreter and a programming language. As a command interpreter, the shell provides the user interface to the rich set of GNU utilities. The programming language features allow these utilities to be combined. Users can create files containing commands, and these become commands themselves. These new commands have the same status as system commands in directories such as /bin, allowing users or groups to establish custom environments to automate their common tasks.

Shells may be used interactively or non-interactively. In interactive mode, they accept input typed from the keyboard. When executing non-interactively, shells execute commands read from a file or a string.

Command Execution

Shells allow execution of GNU commands, both synchronously and asynchronously. The shell waits for synchronous commands to complete before accepting more input; asynchronous commands continue to execute in parallel.

Built-in Commands

Shells provide a small set of built-in commands (builtins) implementing functionality impossible or inconvenient to obtain via separate utilities. For example, cd, break, continue, and exec.

Programming Language

Like any high-level language, the shell provides variables, flow control constructs, quoting, and functions for powerful scripting capabilities.

Interactive Features

Shells offer features geared specifically for interactive use: job control, command line editing, command history and aliases.

special builtin
A shell builtin command that has been classified as special by the POSIX standard.

word
A sequence of characters treated as a unit by the shell. Words may not include unquoted metacharacters.

3. Basic Shell Features

Fundamental shell operations and syntax

Bash is an acronym for 'Bourne-Again SHell'. The Bourne shell is the traditional Unix shell originally written by Stephen Bourne. All of the Bourne shell builtin commands are available in Bash, and the rules for evaluation and quoting are taken from the POSIX specification for the 'standard' Unix shell.

This chapter briefly summarizes the shell's 'building blocks': commands, control structures, shell functions, shell parameters, shell expansions, redirections, which are a way to direct input and output from and to named files, and how the shell executes commands.

3.1 Shell Syntax

When the shell reads input, it proceeds through a sequence of operations. If the input indicates the beginning of a comment, the shell ignores the comment symbol ('#'), and the rest of that line.

Otherwise, roughly speaking, the shell reads its input and divides the input into words and operators, employing the quoting rules to select which meanings to assign various words and characters.

The shell then parses these tokens into commands and other constructs, removes the special meaning of certain words or characters, expands others, redirects input and output as needed, executes the specified command, waits for the command's exit status, and makes that exit status available for further inspection or processing.

3.1.1 Shell Operation

The following is a brief description of the shell's operation when it reads and executes a command. Basically, the shell does the following:

  1. Reads its input from a file, from a string supplied as an argument to the -c invocation option, or from the user's terminal.
  2. Breaks the input into words and operators, obeying the quoting rules. These tokens are separated by metacharacters. This step performs alias expansion.
  3. Parses the tokens into simple and compound commands.
  4. Performs the various shell expansions, breaking the expanded tokens into lists of filenames and commands and arguments.
  5. Performs any necessary redirections and removes the redirection operators and their operands from the argument list.
  6. Executes the command.
  7. Optionally waits for the command to complete and collects its exit status.

3.1.2 Quoting

Quoting is used to remove the special meaning of certain characters or words to the shell. Quoting can be used to disable special treatment for special characters, to prevent reserved words from being recognized as such, and to prevent parameter expansion.

Each of the shell metacharacters has special meaning to the shell and must be quoted if it is to represent itself.

When the command history expansion facilities are being used, the history expansion character, usually '!', must be quoted to prevent history expansion.

There are four quoting mechanisms: the escape character, single quotes, double quotes, and dollar-single quotes.

3.1.2.1 Escape Character

A non-quoted backslash '\' is the Bash escape character. It preserves the literal value of the next character that follows, removing any special meaning it has, with the exception of newline.

echo \$HOME    # Prints literal $HOME
echo \> file   # Prints literal > file
3.1.2.2 Single Quotes

Enclosing characters in single quotes ('') preserves the literal value of each character within the quotes. A single quote may not occur between single quotes, even when preceded by a backslash.

echo '$HOME is my home directory'
# Prints: $HOME is my home directory
3.1.2.3 Double Quotes

Enclosing characters in double quotes ("") preserves the literal value of all characters within the quotes, with the exception of '$', '`', '\', and, when history expansion is enabled, '!'.

echo "$HOME is my home directory"
# Prints: /home/user is my home directory
3.1.2.4 ANSI-C Quoting

Character sequences of the form $'string' are treated as a special kind of single quotes. The sequence expands to string, with backslash-escaped characters replaced as specified by the ANSI C standard.

echo $'Hello\nWorld'    # Prints Hello on one line, World on next
echo $'\t\tIndented'    # Prints with two tab characters

3.2 Shell Commands

A simple shell command such as echo a b c consists of the command itself followed by arguments, separated by spaces. More complex shell commands are composed of simple commands arranged together in a variety of ways.

3.2.5.2 Conditional Constructs

if [ "$1" = "hello" ]; then

3.3 Shell Functions

Shell functions are a way to group commands for later execution using a single name for the group. They are executed just like a "regular" command but in the current shell context.

Function Definition Syntax

Method 1: Traditional
function_name() {
  commands
}
Method 2: With function keyword
function function_name {
  commands
}
Method 3: Full syntax
function function_name() {
  commands
} 
# Simple function
greet() {
    echo "Hello, $1!"
}

# Function with multiple parameters
add_numbers() {
    local num1=$1
    local num2=$2
    local result=$((num1 + num2))
    echo "Result: $result"
    return $result
}

# Function with local variables
process_file() {
    local filename=$1
    local line_count
    
    if [[ -f "$filename" ]]; then
        line_count=$(wc -l < "$filename")
        echo "File '$filename' has $line_count lines"
        return 0
    else
        echo "Error: File '$filename' not found"
        return 1
    fi
}

# Recursive function
factorial() {
    local n=$1
    if (( n <= 1 )); then
        echo 1
    else
        local prev=$(factorial $((n - 1)))
        echo $((n * prev))
    fi
}

# Function usage examples
greet "World"                    # Output: Hello, World!
add_numbers 15 25               # Output: Result: 40
process_file "/etc/passwd"      # Process existing file
result=$(factorial 5)           # result = 120

Function Features

Local Variables

Use local to create function-scoped variables that don't affect the global namespace.

my_func() {
    local temp="function scope"
    echo $temp
}
my_func
echo $temp  # Empty - not in global scope
Return Values

Functions can return exit status codes (0-255) and output results via echo/printf.

is_even() {
    local num=$1
    if (( num % 2 == 0 )); then
        return 0  # Success (even)
    else
        return 1  # Failure (odd)
    fi
}

if is_even 42; then
    echo "42 is even"
fi
Parameter Access

Functions access parameters using $1, $2, etc. $@ and $* provide all parameters.

show_params() {
    echo "Total params: $#"
    echo "All params: $@"
    for param in "$@"; do
        echo "  - $param"
    done
}

show_params one two three

4. Shell Builtin Commands

Built-in commands that are part of the shell itself

Builtin commands are contained within the shell itself. When the name of a builtin command is used as the first word of a simple command, the shell executes the command directly, without invoking another program. Builtin commands are necessary to implement functionality impossible or inconvenient to obtain with separate utilities.

This section briefly describes the builtins which Bash inherits from the Bourne Shell, as well as the builtin commands which are unique to or have been extended in Bash.

Note: Unless otherwise noted, each builtin command documented as accepting options preceded by '-' accepts '--' to signify the end of the options. The :, true, false, and test/[ builtins do not accept options and do not treat '--' specially.

4.1 Bourne Shell Builtins

The following shell builtin commands are inherited from the Bourne Shell. These commands are implemented as specified by the POSIX standard.

: (colon)

: [arguments]

Do nothing beyond expanding arguments and performing redirections. The return status is zero.

# Used as a no-op command
: ${VAR:="default value"}    # Set default value if VAR is unset
: "This is a comment"        # Multi-line comment placeholder

. (source)

. [-p path] filename [arguments]

Read and execute commands from the filename argument in the current shell context.

# Source a configuration file
. ~/.bashrc

# Source with custom path
. -p /etc:/usr/local/etc config_file

# Pass arguments to sourced script
. my_script.sh arg1 arg2

break

break [n]

Exit from a for, while, until, or select loop. If n is supplied, break exits the nth enclosing loop.

for i in {1..10}; do
    if [ $i -eq 5 ]; then
        break    # Exit the loop when i equals 5
    fi
    echo $i
done

cd

cd [-L] [-@] [directory]
cd -P [-e] [-@] [directory]

Change the current working directory to directory. If directory is not supplied, the value of the HOME shell variable is used.

cd /home/user          # Change to specific directory
cd                     # Change to home directory
cd -                   # Change to previous directory
cd -P /path/with/symlinks  # Follow physical path, not symbolic links
  • -L: Follow symbolic links (default behavior)
  • -P: Do not follow symbolic links
  • -e: Exit with error if unable to determine working directory
  • -@: Present extended attributes as a directory (where supported)

continue

continue [n]

Skip the remaining commands in the current iteration of a loop and proceed with the next iteration.

for i in {1..10}; do
    if [ $((i % 2)) -eq 0 ]; then
        continue    # Skip even numbers
    fi
    echo "Odd number: $i"
done

echo

echo [-neE] [arg ...]

Output the args, separated by spaces, followed by a newline.

echo "Hello, World!"           # Basic output
echo -n "No newline"           # Suppress trailing newline
echo -e "Line 1\nLine 2"       # Enable interpretation of backslash escapes
echo -E "Literal \n string"    # Disable interpretation of backslash escapes
  • -n: Do not output the trailing newline
  • -e: Enable interpretation of backslash escapes
  • -E: Disable interpretation of backslash escapes (default)

eval

eval [arg ...]

The args are read and concatenated together into a single command, which is then executed by the shell.

cmd="ls -la"
eval $cmd              # Execute the command stored in variable

# Dynamic variable access
var="HOME"
eval echo \$$var       # Prints the value of $HOME

exec

exec [-cl] [-a name] [command [arguments]]

Replace the shell with the given command without creating a new process.

exec bash              # Replace current shell with new bash instance
exec 3> output.txt      # Redirect file descriptor 3 to output.txt
exec < input.txt       # Redirect stdin from input.txt

exit

exit [n]

Exit the shell with a status of n. If n is omitted, the exit status is that of the last command executed.

exit 0                 # Exit successfully
exit 1                 # Exit with error status
exit                   # Exit with status of last command

export

export [-fn] [name[=word]] ...

Mark each name to be passed to child processes in the environment.

export PATH="/usr/local/bin:$PATH"   # Add to PATH
export EDITOR=vim                    # Set environment variable
export -f my_function               # Export function to child processes

pwd

pwd [-LP]

Print the absolute pathname of the current working directory.

pwd                    # Print current directory
pwd -P                 # Print physical directory (resolve symlinks)
pwd -L                 # Print logical directory (default)

4.2 Bash Builtin Commands

These commands are unique to Bash or have been extended beyond the POSIX specification.

alias

alias [-p] [name[=value] ...]

Define or display aliases. An alias allows a string to be substituted for a word when it is used as the first word of a simple command.

alias ll='ls -la'              # Create alias for long listing
alias grep='grep --color=auto' # Add color to grep
alias                          # List all aliases
unalias ll                     # Remove alias

declare

declare [-aAfFgilnrtux] [-p] [name[=value] ...]

Declare variables and give them attributes, or display variable values and attributes.

declare -i count=10            # Declare integer variable
declare -a array=(1 2 3)       # Declare indexed array
declare -A assoc=([key]="value") # Declare associative array
declare -r readonly_var="fixed"   # Declare read-only variable
declare -f                     # List all functions

local

local [option] [name[=value] ...]

Create local variables within functions. Local variables are only visible within the function and any functions called from it.

my_function() {
    local local_var="This is local"
    local -i count=0
    local -a local_array=(a b c)
    echo "$local_var"
}

printf

printf [-v var] format [arguments]

Write formatted output according to the format string, similar to the C printf function.

printf "Hello, %s!\n" "World"
printf "%d + %d = %d\n" 5 3 8
printf -v result "Formatted: %s" "string"  # Store in variable

read

read [-ers] [-a aname] [-d delim] [-i text] [-n nchars] [-N nchars] [-p prompt] [-t timeout] [-u fd] [name ...]

Read one line from standard input, or from file descriptor fd if the -u option is supplied, and assign words to variable names.

read name              # Read into variable 'name'
read -p "Enter name: " name     # Display prompt
read -s password       # Read silently (no echo)
read -t 10 input       # Timeout after 10 seconds
read -a array          # Read into array

type

type [-aftpP] name [name ...]

Display information about command type - whether it's a builtin, function, alias, or external command.

type ls                # Show what 'ls' is
type -a grep           # Show all instances of 'grep'
type -f my_function    # Show function definition
type -p which          # Show path to external command

3.5 Shell Expansions

Comprehensive guide to all shell expansion mechanisms

Expansion is performed on the command line after it has been split into tokens. Bash performs expansions in a specific order, and understanding this order is crucial for effective shell programming.

Order of Expansions

  1. Brace Expansion
  2. Tilde Expansion, Parameter Expansion, Arithmetic Expansion, and Command Substitution (left-to-right)
  3. Word Splitting
  4. Filename Expansion
  5. Quote Removal (always performed last)

3.5.1 Brace Expansion

Brace expansion generates arbitrary strings sharing a common prefix and suffix. The filenames generated need not exist.

Comma-separated Lists

# Basic brace expansion
echo a{d,c,b}e
# Output: ade ace abe

# With prefix and suffix
echo file{1,2,3}.txt
# Output: file1.txt file2.txt file3.txt

# Nested braces
echo {a,b{1,2},c}
# Output: a b1 b2 c

# Multiple expansions
echo {A,B}{1,2}
# Output: A1 A2 B1 B2

Sequence Expressions

# Numeric sequences
echo {1..5}
# Output: 1 2 3 4 5

# With increment
echo {1..10..2}
# Output: 1 3 5 7 9

# Zero-padded numbers
echo {01..05}
# Output: 01 02 03 04 05

# Reverse order
echo {5..1}
# Output: 5 4 3 2 1

# Character sequences
echo {a..e}

The '$' character introduces parameter expansion, command substitution, or arithmetic expansion. Parameter expansion is the most complex expansion mechanism.

Basic Parameter Expansion

Syntax Description Example

Command substitution allows the output of a command to replace the command itself. Bash supports two forms.

Arithmetic expansion allows the evaluation of an arithmetic expression and substitution of the result.

# Assignment within arithmetic

echo $((x++))                  # 9 (post-increment, x becomes 10)

# Comparison (returns 0 or 1)
echo $((5 > 3))                # 1 (true)

echo $((5 | 3))                # 7 (bitwise OR)
echo $((5 ^ 3))                # 6 (bitwise XOR)
echo $((5 << 1))               # 10 (left shift)
echo $((10 >> 1))              # 5 (right shift)

# For floating point, use bc or awk
area=$(echo "scale=2; 3.14159 * $radius * $radius" | bc)

# Conditional (ternary) operator

# Loop control
for ((i=0; i<10; i++)); do
    echo "Count: $i"
done

#!/bin/bash

# Set strict error handling
set -euo pipefail

# Configuration
readonly SCRIPT_NAME=$(basename "$0")
readonly LOG_FILE="/var/log/${SCRIPT_NAME%.*}.log"
readonly EMAIL_ALERT="admin@example.com"
readonly CPU_THRESHOLD=80
readonly MEMORY_THRESHOLD=85
readonly DISK_THRESHOLD=90

# Logging function
log() {
    local level="$1"
    shift
    printf "[%s] [%s] %s\n" "$(date '+%Y-%m-%d %H:%M:%S')" "$level" "$*" | tee -a "$LOG_FILE"
}

# Check system resources
check_cpu_usage() {
    local cpu_usage
    cpu_usage=$(top -bn1 | grep "Cpu(s)" | sed "s/.*, *\([0-9.]*\)%* id.*/\1/" | awk '{print int(100 - $1)}')
    
    if [[ "$cpu_usage" -gt "$CPU_THRESHOLD" ]]; then
        log "WARN" "High CPU usage: ${cpu_usage}%"
        return 1
    fi
    
    log "INFO" "CPU usage: ${cpu_usage}% (OK)"
    return 0
}

check_memory_usage() {
    local memory_usage
    memory_usage=$(free | grep Mem | awk '{printf "%.0f", $3/$2 * 100.0}')
    
    if [[ "$memory_usage" -gt "$MEMORY_THRESHOLD" ]]; then
        log "WARN" "High memory usage: ${memory_usage}%"
        return 1
    fi
    
    log "INFO" "Memory usage: ${memory_usage}% (OK)"
    return 0
}

check_disk_usage() {
    local disk_usage filesystem
    
    while read -r filesystem usage; do
        usage=${usage%\%}  # Remove % sign
        
        if [[ "$usage" -gt "$DISK_THRESHOLD" ]]; then
            log "WARN" "High disk usage on $filesystem: ${usage}%"
            return 1
        fi
        
        log "INFO" "Disk usage on $filesystem: ${usage}% (OK)"
    done < <(df -h | awk 'NR>1 && $5 ~ /%/ {print $6, $5}')
    
    return 0
}

# Service management
check_critical_services() {
    local services=("ssh" "nginx" "mysql" "fail2ban")
    local failed_services=()
    
    for service in "${services[@]}"; do
        if ! systemctl is-active --quiet "$service" 2>/dev/null; then
            failed_services+=("$service")
            log "ERROR" "Service $service is not running"
        else
            log "INFO" "Service $service is running (OK)"
        fi
    done
    
    if [[ ${#failed_services[@]} -gt 0 ]]; then
        log "ERROR" "Failed services: ${failed_services[*]}"
        return 1
    fi
    
    return 0
}

# Main monitoring function
main() {
    log "INFO" "Starting system check"
    
    local exit_code=0
    local alerts=()
    
    # Run checks
    check_cpu_usage || { alerts+=("CPU"); exit_code=1; }
    check_memory_usage || { alerts+=("Memory"); exit_code=1; }
    check_disk_usage || { alerts+=("Disk"); exit_code=1; }
    check_critical_services || { alerts+=("Services"); exit_code=1; }
    
    # Send alerts if necessary
    if [[ ${#alerts[@]} -gt 0 ]]; then
        local alert_message="System alerts on $(hostname): ${alerts[*]}"
        log "ERROR" "$alert_message"
        
        # Send email alert (if mail is configured)
        if command -v mail &>/dev/null; then
            echo "$alert_message" | mail -s "System Alert: $(hostname)" "$EMAIL_ALERT"
        fi
    else
        log "INFO" "All system checks passed"
    fi
    
    log "INFO" "System check completed"
    exit $exit_code
}

#!/bin/bash
# Application deployment script

# Configuration
readonly APP_NAME="myapp"
readonly DEPLOY_USER="deploy"
readonly REMOTE_HOST="production.example.com"
readonly REMOTE_PATH="/opt/$APP_NAME"
readonly BACKUP_DIR="/opt/backups"
readonly SERVICE_NAME="$APP_NAME"

# Colors for output
readonly RED='\033[0;31m'
readonly GREEN='\033[0;32m'
readonly YELLOW='\033[1;33m'
readonly NC='\033[0m' # No Color

# Logging functions
info() { printf "${GREEN}[INFO]${NC} %s\n" "$*"; }
warn() { printf "${YELLOW}[WARN]${NC} %s\n" "$*"; }
error() { printf "${RED}[ERROR]${NC} %s\n" "$*" >&2; }

# Deployment functions
check_prerequisites() {
    info "Checking prerequisites..."
    
    # Check if git is available
    command -v git >/dev/null 2>&1 || { error "Git is required but not installed"; return 1; }
    
    # Check if we can connect to remote host
    if ! ssh -o ConnectTimeout=5 "$DEPLOY_USER@$REMOTE_HOST" "echo 'Connection test successful'" >/dev/null 2>&1; then
        error "Cannot connect to remote host: $REMOTE_HOST"
        return 1
    fi
    
    # Check if local build is successful
    if ! npm test >/dev/null 2>&1; then
        error "Tests failed, deployment aborted"
        return 1
    fi
    
    info "Prerequisites check passed"
    return 0
}

create_backup() {
    info "Creating backup of current deployment..."
    
    local timestamp=$(date +%Y%m%d_%H%M%S)
    local backup_name="${APP_NAME}_backup_${timestamp}"
    
    ssh "$DEPLOY_USER@$REMOTE_HOST" "
        if [[ -d '$REMOTE_PATH' ]]; then
            sudo mkdir -p '$BACKUP_DIR'
            sudo tar -czf '$BACKUP_DIR/$backup_name.tar.gz' -C '$(dirname "$REMOTE_PATH")' '$(basename "$REMOTE_PATH")'
            echo 'Backup created: $BACKUP_DIR/$backup_name.tar.gz'
        else
            echo 'No existing deployment found, skipping backup'
        fi
    "
}

deploy_application() {
    info "Deploying application..."
    
    # Create deployment package
    local temp_dir=$(mktemp -d)
    local package_name="${APP_NAME}_$(git rev-parse --short HEAD)_$(date +%Y%m%d_%H%M%S).tar.gz"
    
    info "Creating deployment package: $package_name"
    
    # Build and package application
    npm run build
    tar --exclude='.git' --exclude='node_modules' --exclude='*.log' \
        -czf "$temp_dir/$package_name" -C . .
    
    # Upload package
    info "Uploading package to remote host..."
    scp "$temp_dir/$package_name" "$DEPLOY_USER@$REMOTE_HOST:/tmp/"
    
    # Deploy on remote host
    info "Extracting and installing on remote host..."
    ssh "$DEPLOY_USER@$REMOTE_HOST" "
        # Stop service
        sudo systemctl stop '$SERVICE_NAME' 2>/dev/null || true
        
        # Create/update application directory
        sudo mkdir -p '$REMOTE_PATH'
        sudo rm -rf '$REMOTE_PATH'/*
        
        # Extract new version
        cd '$REMOTE_PATH'
        sudo tar -xzf '/tmp/$package_name'
        sudo chown -R '$DEPLOY_USER:$DEPLOY_USER' '$REMOTE_PATH'
        
        # Install dependencies
        npm install --production
        
        # Update configuration
        if [[ -f '/etc/$APP_NAME/config.json' ]]; then
            ln -sf '/etc/$APP_NAME/config.json' '$REMOTE_PATH/config.json'
        fi
        
        # Start service
        sudo systemctl start '$SERVICE_NAME'
        sudo systemctl enable '$SERVICE_NAME'
        
        # Cleanup
        rm -f '/tmp/$package_name'
    "
    
    # Cleanup local temp directory
    rm -rf "$temp_dir"
    
    info "Application deployed successfully"
}

verify_deployment() {
    info "Verifying deployment..."
    
    # Check if service is running
    if ssh "$DEPLOY_USER@$REMOTE_HOST" "sudo systemctl is-active --quiet '$SERVICE_NAME'"; then
        info "Service is running"
    else
        error "Service failed to start"
        return 1
    fi
    
    # Check if application responds
    local app_url="http://$REMOTE_HOST:3000/health"
    if curl -f -s "$app_url" >/dev/null; then
        info "Application health check passed"
    else
        warn "Application health check failed or endpoint not available"
    fi
    
    # Display service status
    ssh "$DEPLOY_USER@$REMOTE_HOST" "sudo systemctl status '$SERVICE_NAME' --no-pager -l"
    
    info "Deployment verification completed"
}

rollback_deployment() {
    warn "Rolling back deployment..."
    
    # Find latest backup
    local latest_backup
    latest_backup=$(ssh "$DEPLOY_USER@$REMOTE_HOST" "ls -1t '$BACKUP_DIR'/${APP_NAME}_backup_*.tar.gz 2>/dev/null | head -1")
    
    if [[ -z "$latest_backup" ]]; then
        error "No backup found for rollback"
        return 1
    fi
    
    info "Rolling back to: $latest_backup"
    
    ssh "$DEPLOY_USER@$REMOTE_HOST" "
        # Stop current service
        sudo systemctl stop '$SERVICE_NAME' 2>/dev/null || true
        
        # Remove current deployment
        sudo rm -rf '$REMOTE_PATH'/*
        
        # Restore from backup
        cd '$REMOTE_PATH'
        sudo tar -xzf '$latest_backup'
        sudo chown -R '$DEPLOY_USER:$DEPLOY_USER' '$REMOTE_PATH'
        
        # Start service
        sudo systemctl start '$SERVICE_NAME'
    "
    
    info "Rollback completed"
}

# Main deployment process
main() {
    local action="${1:-deploy}"
    
    case "$action" in
        "deploy")
            check_prerequisites && \
            create_backup && \
            deploy_application && \
            verify_deployment
            ;;
        "rollback")
            rollback_deployment
            ;;
        *)
            echo "Usage: $0 [deploy|rollback]"
            exit 1
            ;;
    esac
}

main "$@"
