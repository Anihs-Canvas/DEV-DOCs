9. Complete Script Template

#!/usr/bin/env bash
#===============================================================================
# Script Name: template.sh
# Description: Production-ready Bash script template
# Author:      Your Name
# Date:        $(date +%Y-%m-%d)
# Version:     1.0.0
#===============================================================================

set -euo pipefail
IFS=$'\n\t'

#-------------------------------------------------------------------------------
# Constants
#-------------------------------------------------------------------------------
readonly SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
readonly SCRIPT_NAME="$(basename "$0")"
readonly VERSION="1.0.0"

#-------------------------------------------------------------------------------
# Colors (optional)
#-------------------------------------------------------------------------------
readonly RED='\033[0;31m'
readonly GREEN='\033[0;32m'
readonly YELLOW='\033[1;33m'
readonly NC='\033[0m' # No Color

#-------------------------------------------------------------------------------
# Logging Functions
#-------------------------------------------------------------------------------
log()   { printf "${GREEN}[INFO]${NC} %s\n" "$*"; }
warn()  { printf "${YELLOW}[WARN]${NC} %s\n" "$*" >&2; }
error() { printf "${RED}[ERROR]${NC} %s\n" "$*" >&2; }
die()   { error "$*"; exit 1; }

#-------------------------------------------------------------------------------
# Cleanup Function
#-------------------------------------------------------------------------------
cleanup() {
    local exit_code=$?
    # Add cleanup tasks here (remove temp files, etc.)
    exit $exit_code
}
trap cleanup EXIT

#-------------------------------------------------------------------------------
# Usage
#-------------------------------------------------------------------------------
usage() {
    cat <<EOF
Usage: $SCRIPT_NAME [OPTIONS] [ARGUMENTS]

Description:
    Brief description of what this script does.

Options:
    -h, --help      Show this help message
    -v, --version   Show version
    -d, --debug     Enable debug mode
    -q, --quiet     Suppress output

Examples:
    $SCRIPT_NAME -h
    $SCRIPT_NAME --debug input.txt
EOF
}

#-------------------------------------------------------------------------------
# Parse Arguments
#-------------------------------------------------------------------------------
parse_args() {
    while [[ $# -gt 0 ]]; do
        case "$1" in
            -h|--help)    usage; exit 0 ;;
            -v|--version) echo "$VERSION"; exit 0 ;;
            -d|--debug)   set -x; shift ;;
            -q|--quiet)   QUIET=1; shift ;;
            --)           shift; break ;;
            -*)           die "Unknown option: $1" ;;
            *)            ARGS+=("$1"); shift ;;
        esac
    done
}

#-------------------------------------------------------------------------------
# Main Function
#-------------------------------------------------------------------------------
main() {
    parse_args "$@"
    
    log "Starting $SCRIPT_NAME v$VERSION"
    
    # Your main logic here
    
    log "Completed successfully"
}

# Run main function
main "$@"
Output
[INFO] Starting template.sh v1.0.0
[INFO] Completed successfully

1. Introduction

Bash is the shell, or command language interpreter, for the GNU operating system. The name is an acronym for the Bourne-Again SHell, a pun on Stephen Bourne, the author of the direct ancestor of the current Unix shell sh.

Bash is largely compatible with sh and incorporates useful features from the Korn shell ksh and the C shell csh. It is intended to be a conformant implementation of the IEEE POSIX Shell and Tools portion of the IEEE POSIX specification (IEEE Standard 1003.1).

Key Features
• Command-line editing
• Unlimited command history
• Job control
• Shell functions and aliases
• Indexed and associative arrays
• Integer arithmetic in any base from 2 to 64

1.1 What is Bash?

Full Description

Bash is the GNU Project's shell—a command processor that typically runs in a text window where the user types commands that cause actions. Bash can also read and execute commands from a file, called a shell script.

Syntax

bash [options] [command_string | file]

Parameters

-c string — Read and execute commands from string after processing the options

-i — Make the shell interactive

-l — Make bash act as if invoked as a login shell

-r — Make the shell restricted

-s — Read commands from standard input

-v — Print shell input lines as they are read

-x — Print commands and their arguments as they are executed

Return Value

Returns the exit status of the last command executed, or 0 if no command was executed. On error, returns a non-zero value.

Examples

bash --version

Hello from Bash

Example 3: Run a Script File

bash test.sh

# Start bash in verbose mode (prints commands as read)

A shell is a macro processor that executes commands. It provides an interface between the user and the operating system kernel. Shells can be interactive (reading commands from user input) or non-interactive (executing commands from a script).

3. Basic Shell Features

Bash is an acronym for 'Bourne-Again SHell'. This section describes the basic shell features: syntax, commands, functions, parameters, expansions, redirections, and scripts.

3.1 Shell Syntax

Full Description

When the shell reads input, it proceeds through a sequence of operations: reads input, breaks it into words and operators, parses tokens into commands, and performs expansions.

Shell Input Processing

Processing Order
1. Read input (line or script)
2. Break into tokens (words and operators)
3. Parse tokens into simple and compound commands
4. Perform shell expansions
5. Perform redirections
6. Execute the command
7. Wait for command to complete and collect exit status

3.1.1 Shell Operation

Full Description

The shell reads and executes commands. When invoked, it reads its startup files, then enters a read-execute loop.

Operation Steps

1. Reads input from file, string (-c), or terminal

2. Breaks input into words and operators following quoting rules

3. Parses tokens into simple and compound commands

4. Performs shell expansions

5. Performs necessary redirections

6. Executes the command

7. Optionally waits for command to complete and collects exit status

Examples

Example 1: Basic Command Execution

# Shell reads, parses, and executes
echo "Hello World"
Output
Hello World

Example 2: Multiple Commands on One Line

# Semicolon separates commands
echo "First"; echo "Second"; echo "Third"
Output
First Second Third

Example 3: Command with Expansion

# Shell expands variables before execution
name="Alice"
echo "Hello, $name"
Output
Hello, Alice

Example 4: Command with Redirection

# Shell sets up redirection before executing
echo "Log entry" > logfile.txt
cat logfile.txt
Output
Log entry

Example 5: Background Execution

# & causes shell not to wait for completion
sleep 2 &
echo "Command started in background"
echo "Process ID: $!"
Output

3.1.2 Quoting

Full Description

Quoting removes the special meaning of certain characters or words. It can disable special treatment of special characters, prevent reserved words from being recognized, and prevent parameter expansion.

Quoting Mechanisms

"..."Double quotes - preserves literal value except $ ` \ !

$"..."Locale translation - translates string according to locale

Full Description

A non-quoted backslash \ is the Bash escape character. It preserves the literal value of the next character that follows, with the exception of newline.

Syntax

\character

3.1.2.2 Single Quotes

Full Description

Enclosing characters in single quotes '...' preserves the literal value of each character within the quotes. A single quote cannot occur between single quotes, even when preceded by a backslash.

Syntax

'string'

Examples

Example 1: Prevent Variable Expansion

# Variables are NOT expanded in single quotes
name="John"
echo 'Hello, $name'
Output
Hello, $name

Example 2: Preserve Special Characters

# All special characters are literal
echo 'Price: $100 | Status: OK && Done!'
Output
Price: $100 | Status: OK && Done!

Example 3: Include Double Quotes

# Double quotes are literal inside single quotes
echo 'He said "Hello" to everyone'
Output
He said "Hello" to everyone

Example 4: Backslash is Literal

# Backslash has no special meaning
echo 'C:\Users\Documents\file.txt'
Output
C:\Users\Documents\file.txt

Example 5: Include Single Quote (Concatenation)

# Use concatenation to include single quote
echo 'It'\''s a beautiful day'
Output
It's a beautiful day

3.1.2.3 Double Quotes

Full Description

Enclosing characters in double quotes "..." preserves the literal value of all characters except $, `, \, and ! (when history expansion is enabled).

Syntax

"string"

Special Characters in Double Quotes

$ — Parameter expansion occurs

`...` — Command substitution occurs

\ — Escapes only $ ` " \ newline

Examples

Example 1: Variable Expansion

# Variables ARE expanded in double quotes
name="Alice"
echo "Hello, $name"
Output
Hello, Alice

Example 2: Command Substitution

# Command substitution works
echo "Today is $(date +%A)"
Output
Today is Saturday

Example 3: Preserve Whitespace

# Whitespace is preserved
text="  spaces   preserved  "
echo "$text"
Output
spaces preserved

Example 4: Include Single Quotes

# Single quotes are literal inside double quotes
echo "It's easy"
Output
It's easy

Example 5: Escape Dollar Sign

# Escape $ to prevent expansion
price=50
echo "Cost: \$${price}"
Output
Cost: $50

Full Description

Words of the form $'string' are treated specially. The word expands to string, with backslash-escaped characters replaced as specified by the ANSI C standard.

Syntax

$'string'

Escape Sequences

\tHorizontal tab
\rCarriage return
\\Backslash
\'Single quote
\aAlert (bell)

\xHHHex value HH

# Insert newline in string
echo $'Line 1\nLine 2\nLine 3'
Output
Line 1 Line 2 Line 3

# Insert tabs
echo $'Name\tAge\tCity'

# Include single quote easily
