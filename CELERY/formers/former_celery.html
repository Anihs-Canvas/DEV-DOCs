Introduction to Celery in Django

Celery is a distributed task queue system for Python that allows you to execute tasks asynchronously outside of the HTTP request-response cycle. It's particularly useful in Django applications for handling time-consuming operations that would otherwise block the web server and degrade user experience.

ˇ

Asynchronous Processing

Execute long-running tasks without blocking HTTP responses, improving application responsiveness and user experience.

=Č

Scalability

Distribute tasks across multiple workers and machines to handle high workloads efficiently.

=

Reliability

Tasks can be retried, scheduled, and monitored with built-in failure handling mechanisms.

<Ż Why Use Celery with Django?

=ˇ Key Benefits
  • Asynchronous Processing: Execute long-running tasks without blocking HTTP responses
  • Scalability: Distribute tasks across multiple workers and machines
  • Reliability: Tasks can be retried, scheduled, and monitored
  • Performance: Improves application responsiveness by offloading heavy operations
  • Background Jobs: Handle periodic tasks, email sending, data processing, etc.

>ŕ Core Concepts

=Ë

Task

A unit of work that can be executed asynchronously by Celery workers.

=w

Worker

A process that executes tasks. Multiple workers can run simultaneously.

=î

Broker

Message transport system (Redis, RabbitMQ, etc.) that queues tasks.

=ľ

Result Backend

Storage system for task results and status information.

đ

Beat

Scheduler for periodic tasks, like cron jobs.

<× Architecture Overview

=Ę Celery Architecture Flow 
Django App ’ Celery ’ Message Broker ’ Celery Workers ’ Result Backend
     “           “           “              “              “
  Web Request  Task Queue  Redis/RMQ   Task Execution   Task Results

The architecture consists of several components working together to process tasks asynchronously. When a Django view needs to perform a long-running operation, it sends a task to Celery, which queues it in the message broker. Workers then pick up tasks from the queue and execute them, storing results in the result backend.

=Ë Task Definition and Registration

@app.task Decorator

Full Description: The @app.task decorator is the primary way to define a Celery task. It converts a regular Python function into a Celery task that can be executed asynchronously by workers. The decorator registers the function with the Celery application and provides various configuration options for task behavior.

Basic Task Syntax 
@app.task(bind=False, name=None, base=None, retry=True, max_retries=3, default_retry_delay=60)
def task_function(arg1, arg2, kwarg1=None):
    # Task logic here
    return result
™ Parameters Description
  • bind: Boolean indicating whether the task should be bound to the task instance (allows access to self)
  • name: Custom name for the task (defaults to module.function_name)
  • base: Base class for the task
  • retry: Whether the task should retry on failure
  • max_retries: Maximum number of retry attempts
  • default_retry_delay: Default delay between retries in seconds

Example 1: Simple Job Notification Task

tasks.py 
from celery import Celery
from django.core.mail import send_mail
from datetime import datetime
from myapp.models import JobPost, Author, User

app = Celery('job_portal')

@app.task
def send_job_notification(job_post_id):
    """Send email notification when a new job is posted"""
    try:
        job_post = JobPost.objects.get(id=job_post_id)
        author = job_post.author
        user = author.user
        
        subject = f"New Job Posted: {job_post.title}"
        message = f"""
        Dear {user.first_name},
        
        Your job posting "{job_post.title}" has been successfully created.
        
        Details:
        - Title: {job_post.title}
        - Description: {job_post.description}
        - Salary: ${job_post.salary:,}
        - Location: {job_post.location.city}, {job_post.location.state}
        - Skills Required: {', '.join([skill.name for skill in job_post.skills.all()])}
        
        Best regards,
        Job Portal Team
        """
        
        result = send_mail(
            subject=subject,
            message=message,
            from_email='noreply@jobportal.com',
            recipient_list=[user.email],
            fail_silently=False
        )
        
        print(f"Email sent successfully to {user.email}")
        return {
            'status': 'success',
            'recipient': user.email,
            'job_title': job_post.title,
            'sent_at': str(datetime.now())
        }
        
    except JobPost.DoesNotExist:
        print(f"JobPost with ID {job_post_id} not found")
        return {'status': 'error', 'message': 'Job post not found'}
    except Exception as e:
        print(f"Error sending notification: {str(e)}")
        return {'status': 'error', 'message': str(e)}

# Usage
result = send_job_notification.delay(1)
print(f"Task ID: {result.id}")
print(f"Task Status: {result.status}")

Example 2: Bulk Skills Update Task

Bulk Update Task 
@app.task(bind=True, max_retries=3)
def bulk_update_job_skills(self, job_ids, skill_names):
    """Update skills for multiple job posts"""
    try:
        updated_jobs = []
        skills_objs = Skills.objects.filter(name__in=skill_names)
        
        print(f"Found {skills_objs.count()} skills to update")
        print(f"Skills: {[skill.name for skill in skills_objs]}")
        
        for job_id in job_ids:
            job_post = JobPost.objects.get(id=job_id)
            
            # Clear existing skills and add new ones
            job_post.skills.clear()
            job_post.skills.add(*skills_objs)
            
            updated_jobs.append({
                'job_id': job_id,
                'title': job_post.title,
                'skills_added': [skill.name for skill in skills_objs],
                'total_skills': job_post.skills.count()
            })
            
            print(f"Updated job {job_id}: {job_post.title}")
            
        return {
            'status': 'success',
            'updated_count': len(updated_jobs),
            'jobs': updated_jobs,
            'processed_at': str(datetime.now())
        }
        
    except Exception as exc:
        print(f"Task failed: {str(exc)}")
        raise self.retry(exc=exc, countdown=60, max_retries=3)

# Usage
result = bulk_update_job_skills.delay([1, 2, 3], ['Python', 'Django', 'JavaScript'])
print(f"Bulk update task started: {result.id}")

=€ Task Execution Methods

delay() vs apply_async()

Celery provides two main methods for executing tasks asynchronously: delay() and apply_async(). While delay() is simpler to use, apply_async() offers more control over task execution.

Task Execution Comparison 
# Using delay() - Simple approach
result = send_job_notification.delay(job_post_id=1)

# Using apply_async() - Advanced approach with options
result = send_job_notification.apply_async(
    args=[1],  # Positional arguments
    kwargs={},  # Keyword arguments
    countdown=60,  # Delay execution by 60 seconds
    eta=datetime.now() + timedelta(minutes=30),  # Execute at specific time
    expires=3600,  # Task expires after 1 hour
    retry=True,  # Enable retries
    retry_policy={
        'max_retries': 3,
        'interval_start': 0,
        'interval_step': 0.2,
        'interval_max': 0.2,
    },
    routing_key='high_priority',  # Route to specific queue
    priority=9,  # Task priority (0-9, higher is better)
    queue='email_queue'  # Specific queue name
)
  Best Practice Tip

Use delay() for simple task execution and apply_async() when you need advanced scheduling, routing, or retry policies.

from celery import shared_task
from django.core.mail import send_mail
from django.template.loader import render_to_string
from django.utils import timezone
from datetime import datetime, timedelta
from .models import JobPost, Skills, Author, User
import logging

logger = logging.getLogger(__name__)

@shared_task(bind=True, max_retries=3)
def send_job_notification(self, job_post_id):
    """Send email notification when a new job is posted"""
    try:
        job_post = JobPost.objects.select_related('author__user', 'location').prefetch_related('skills').get(id=job_post_id)
        user = job_post.author.user
        
        # Prepare email context
        context = {
            'user': user,
            'job_post': job_post,
            'skills': job_post.skills.all(),
            'location': job_post.location,
        }
        
        # Render email template
        subject = f"New Job Posted: {job_post.title}"
        html_message = render_to_string('emails/job_notification.html', context)
        plain_message = render_to_string('emails/job_notification.txt', context)
        
        # Send email
        result = send_mail(
            subject=subject,
            message=plain_message,
            html_message=html_message,
            from_email='noreply@jobportal.com',
            recipient_list=[user.email],
            fail_silently=False
        )
        
        logger.info(f"Email sent successfully to {user.email} for job {job_post.title}")
        return {
            'status': 'success',
            'recipient': user.email,
            'job_title': job_post.title,
            'sent_at': timezone.now().isoformat()
        }
        
    except JobPost.DoesNotExist:
        logger.error(f"JobPost with ID {job_post_id} not found")
        return {'status': 'error', 'message': 'Job post not found'}
    except Exception as exc:
        logger.error(f"Error sending notification: {str(exc)}")
        raise self.retry(exc=exc, countdown=60 * (self.request.retries + 1))

@shared_task
def bulk_update_job_skills(job_ids, skill_names):
    """Update skills for multiple job posts"""
    try:
        updated_jobs = []
        skills_objs = Skills.objects.filter(name__in=skill_names)
        job_posts = JobPost.objects.filter(id__in=job_ids)
        
        logger.info(f"Updating {len(job_ids)} jobs with {len(skill_names)} skills")
        
        for job_post in job_posts:
            # Clear existing skills and add new ones
            job_post.skills.set(skills_objs)
            
            updated_jobs.append({
                'job_id': job_post.id,
                'title': job_post.title,
                'skills_added': list(skills_objs.values_list('name', flat=True)),
                'total_skills': job_post.skills.count()
            })
            
        logger.info(f"Successfully updated {len(updated_jobs)} jobs")
        return {
            'status': 'success',
            'updated_count': len(updated_jobs),
            'jobs': updated_jobs,
            'processed_at': timezone.now().isoformat()
        }
        
    except Exception as exc:
        logger.error(f"Bulk update failed: {str(exc)}")
        raise exc

@shared_task
def cleanup_expired_jobs():
    """Remove expired job postings"""
    try:
        current_date = timezone.now().date()
        expired_jobs = JobPost.objects.filter(expiry__lt=current_date)
        count = expired_jobs.count()
        
        if count > 0:
            expired_jobs.delete()
            logger.info(f"Cleaned up {count} expired job postings")
        
        return {
            'status': 'success',
            'cleaned_count': count,
            'cleaned_at': timezone.now().isoformat()
        }
        
    except Exception as exc:
        logger.error(f"Cleanup task failed: {str(exc)}")
        raise exc

@shared_task
def generate_job_report():
    """Generate daily job statistics report"""
    try:
        today = timezone.now().date()
        yesterday = today - timedelta(days=1)
        
        # Get statistics
        total_jobs = JobPost.objects.count()
        new_jobs_today = JobPost.objects.filter(date__date=today).count()
        jobs_by_type = JobPost.objects.values('type').annotate(count=models.Count('id'))
        
        report_data = {
            'date': today.isoformat(),
            'total_jobs': total_jobs,
            'new_jobs_today': new_jobs_today,
            'jobs_by_type': list(jobs_by_type),
            'generated_at': timezone.now().isoformat()
        }
        
        logger.info(f"Generated job report for {today}")
        return report_data
        
    except Exception as exc:
        logger.error(f"Report generation failed: {str(exc)}")
        raise exc

Task Design

  • Keep tasks idempotent (can be safely retried)
  • Use database transactions carefully
  • Minimize task payload size
  • Use select_related() and prefetch_related()

Worker Configuration

  • Set appropriate concurrency levels
  • Use worker pools (prefork, eventlet, gevent)
  • Configure memory limits per worker
  • Monitor worker performance
=Ę

Queue Management

  • Use multiple queues for different task types
  • Set queue priorities appropriately
  • Monitor queue lengths
  • Implement dead letter queues

from django.test import TestCase, override_settings
from unittest.mock import patch, MagicMock
from celery import current_app
from myapp.tasks import send_job_notification, bulk_update_job_skills
from myapp.models import JobPost, Skills, Author, User

# Test with eager execution (synchronous)
@override_settings(CELERY_TASK_ALWAYS_EAGER=True)
class CeleryTaskTests(TestCase):
    
    def setUp(self):
        # Create test data
        self.user = User.objects.create(
            username='testuser',
            email='test@example.com',
            first_name='Test'
        )
        self.author = Author.objects.create(
            user=self.user,
            company='Test Company',
            designation='Test Role'
        )
        
    def test_send_job_notification_success(self):

        # Create test job post
        job_post = JobPost.objects.create(
            title='Test Job',
            description='Test Description',
            salary=50000,
            author=self.author
        )
        
        with patch('myapp.tasks.send_mail') as mock_send_mail:
            mock_send_mail.return_value = True
            
            result = send_job_notification.delay(job_post.id)
            
            self.assertEqual(result.get()['status'], 'success')
            mock_send_mail.assert_called_once()
    
    def test_send_job_notification_nonexistent_job(self):
        """Test notification for non-existent job"""
        result = send_job_notification.delay(999)
        self.assertEqual(result.get()['status'], 'error')
    
    @patch('myapp.tasks.logger')
    def test_bulk_update_job_skills(self, mock_logger):

        # Create test data
        skill1 = Skills.objects.create(name='Python')
        skill2 = Skills.objects.create(name='Django')
        
        job_post = JobPost.objects.create(
            title='Test Job',
            description='Test Description',
            salary=50000,
            author=self.author
        )

        self.assertEqual(result_data['updated_count'], 1)

# Test with actual Celery worker
class CeleryIntegrationTests(TestCase):
    
    def test_task_routing(self):
        """Test that tasks are routed to correct queues"""
        with current_app.pool.acquire(block=True) as connection:
            result = send_job_notification.apply_async(
                args=[1],
                connection=connection
            )
            
            # Check task is in the correct queue
            self.assertIn('email_queue', str(result.backend))
    
    def test_task_retry_mechanism(self):
        """Test task retry behavior"""
        with patch('myapp.models.JobPost.objects.get') as mock_get:
            mock_get.side_effect = Exception('Database error')
            
            result = send_job_notification.apply_async(args=[1])
            
            # Task should retry and eventually fail
            self.assertEqual(result.state, 'RETRY')
