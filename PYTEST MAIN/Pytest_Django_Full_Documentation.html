📊 Constants

Pytest constants provide version information and special parameters that can be used throughout your test suite. These constants help you write version-aware tests and control test naming behavior.

pytest.__version__

Full Description

The pytest.__version__ constant returns the current installed version of pytest as a string. This is useful for checking pytest version compatibility in your test suite or when debugging test failures that might be version-specific. The version string follows semantic versioning (MAJOR.MINOR.PATCH) format.

Syntax

import pytest
version = pytest.__version__

Parameters Description

This constant does not accept any parameters. It is a read-only string attribute that contains the version information.

Attributes

Type: str

Format: MAJOR.MINOR.PATCH (e.g., "7.0.0")

Access: Read-only attribute

Module: pytest

Methods

This constant has no methods. It is a simple string constant. However, you can use standard Python string methods on it such as:

  • .split('.') - Split version into components
  • .startswith() - Check version prefix
  • comparison operators - Compare versions (use packaging.version for proper comparison)

Return Value

Returns: A string representing the current pytest version in the format "MAJOR.MINOR.PATCH"

Example return values: "7.0.0", "7.4.3", "8.0.0rc1"

pytest.HIDDEN_PARAM

Full Description

The pytest.HIDDEN_PARAM constant (added in pytest 8.4) is a special marker that can be passed to the ids parameter of pytest.mark.parametrize() or to the id parameter of pytest.param() to hide a parameter set from the test name. This is useful when you want to run parametrized tests but don't want certain parameter values to appear in the test output. It can only be used at most once per test since test names need to be unique.

Syntax

import pytest

# Use in parametrize
@pytest.mark.parametrize("param", [value1, value2], ids=[pytest.HIDDEN_PARAM, "visible"])
def test_function(param):
    pass

# Use in param
@pytest.mark.parametrize("value", [
    pytest.param(data, id=pytest.HIDDEN_PARAM),
    pytest.param(data2, id="named")
])

Parameters Description

This constant does not accept parameters. It is used as a value assignment to hide parameter identifiers in test names.

  • Usage context: Only in ids parameter of parametrize or id parameter of pytest.param()
  • Limitation: Can only be used once per test (test names must be unique)
  • Purpose: Hide sensitive or verbose parameter values from test output

Attributes

Type: Special marker constant

Added: pytest 8.4

Module: pytest

Usage: Test parametrization and naming control

Methods

This constant has no methods. It serves as a special marker value that pytest recognizes during test collection and naming.

Return Value

Returns: N/A - This is a marker constant used for test naming control, not a callable function.

pytest.version_tuple

Full Description

The pytest.version_tuple constant (added in pytest 7.0) returns the current pytest version as a tuple of integers and strings, making version comparisons more convenient than string parsing. The tuple format is (major, minor, patch) for stable releases, and (major, minor, prerelease_string) for pre-releases. This allows for easy programmatic version checking using standard Python tuple comparison operators.

Syntax

import pytest
version = pytest.version_tuple

# Example returns:
# (7, 4, 3) for stable version 7.4.3
# (7, 0, '0rc1') for pre-release 7.0.0rc1

Parameters Description

This constant does not accept any parameters. It is a read-only tuple attribute containing version components.

  • Stable versions: Returns tuple of three integers (major, minor, patch)
  • Pre-releases: Last component is a string (e.g., '0rc1', '1a1', '2b3')
  • Comparison: Can be compared directly using Python operators (<, <=, ==, >=, >)

Attributes

Type: tuple

Format: (major: int, minor: int, patch: int | str)

Added: pytest 7.0

Access: Read-only attribute

Module: pytest

Methods

This constant has no methods. It is a tuple, so standard tuple operations apply:

  • Indexing: version_tuple[0] for major version
  • Unpacking: major, minor, patch = version_tuple
  • Slicing: version_tuple[:2] for major.minor
  • Comparison: Direct comparison with other tuples

Return Value

Returns: A tuple representing the version

Stable version example: (7, 4, 3) for pytest 7.4.3

Pre-release example: (7, 0, '0rc1') for pytest 7.0.0rc1

pytest.approx

Full Description

The pytest.approx function provides a convenient way to compare floating-point numbers with a tolerance, avoiding issues with floating-point precision. It's especially useful when testing calculations involving money, percentages, or any mathematical operations that may produce small rounding errors. In Django applications, it's commonly used to validate salary calculations, price computations, statistical aggregations, and financial data where exact floating-point comparison would be inappropriate.

Syntax

pytest.approx(expected, rel=None, abs=None, nan_ok=False)

Parameters Description

  • expected (float | Sequence[float] | Mapping[str, float]): The expected value or collection of values to compare against.
  • rel (float | None): Relative tolerance (as a fraction). Default is 1e-6 (0.0001%).
  • abs (float | None): Absolute tolerance. Default is 1e-12.
  • nan_ok (bool): If True, NaN values are considered equal. Default is False.

Return Value

Returns: An ApproxScalar, ApproxSequencelike, or ApproxMapping object that can be compared with actual values using standard comparison operators.

pytest.fail

Full Description

The pytest.fail function explicitly fails a test with a given message. It's useful when you need to fail a test based on complex conditions that can't be easily expressed with simple assertions, or when you want to provide detailed failure messages. In Django applications, it's commonly used for conditional test failures, validation of business logic constraints, and providing detailed error context when tests encounter unexpected states.

Syntax

pytest.fail(msg='', pytrace=True)

Parameters Description

  • msg (str): The failure message to display. Default is empty string.
  • pytrace (bool): Whether to show Python traceback. Default is True.

Return Value

Returns: NoReturn - This function raises a Failed exception and never returns.

pytest.skip

Full Description

The pytest.skip function unconditionally skips a test execution with an optional reason. It's useful when you need to skip tests based on runtime conditions, missing dependencies, or specific environment requirements. In Django applications, it's commonly used to skip tests when certain database features are unavailable, optional packages are missing, or environment-specific conditions aren't met.

Syntax

pytest.skip(msg='', allow_module_level=False)

Parameters Description

  • msg (str): The reason for skipping the test. Default is empty string.
  • allow_module_level (bool): If True, allows skipping at module level. Default is False.

Return Value

Returns: NoReturn - This function raises a Skipped exception and never returns.

pytest.importorskip

Full Description

The pytest.importorskip function imports a module and skips the test if the import fails or the module version doesn't meet requirements. It's useful for handling optional dependencies gracefully. In Django applications, it's used to skip tests when optional packages like Pillow, Celery, or Redis are not installed.

Syntax

pytest.importorskip(modname, minversion=None, reason=None)

Parameters Description

  • modname (str): Name of the module to import.
  • minversion (str | None): Minimum required version. Default is None.
  • reason (str | None): Custom skip reason. Default is None.

Return Value

Returns: The imported module if successful, otherwise raises Skipped.

Examples

pytest.raises

Full Description

The pytest.raises context manager is used to assert that a specific exception is raised within a code block. It's one of the most commonly used pytest functions for testing error handling. In Django applications, it's essential for validating model constraints, form validation errors, and business logic exceptions.

Syntax

with pytest.raises(expected_exception, match=None) as excinfo:
    # code that should raise exception

Parameters Description

  • expected_exception (Type[BaseException] | Tuple): The exception class(es) expected to be raised.
  • match (str | Pattern | None): Optional regex pattern that the exception message must match.

Return Value

Returns: ExceptionInfo object containing information about the raised exception.

pytest.warns

Full Description

The pytest.warns context manager is used to assert that a specific warning is issued within a code block. It's similar to pytest.raises but for warnings instead of exceptions. In Django applications, it's useful for testing deprecation warnings, configuration warnings, and other non-fatal issues.

Syntax

with pytest.warns(expected_warning, match=None) as warning_list:
    # code that should issue warning

Parameters Description

  • expected_warning (Type[Warning] | Tuple): The warning class(es) expected to be issued.
  • match (str | Pattern | None): Optional regex pattern that the warning message must match.

Return Value

Returns: WarningsRecorder object containing information about the issued warnings.

pytest.xfail

Full Description

The pytest.xfail function marks a test as "expected to fail" during execution. Unlike skip, the test still runs but failures are not reported as test failures. If the test unexpectedly passes, it's marked as "XPASS". This is useful for known bugs or features under development in Django applications.

Syntax

pytest.xfail(reason='')

Parameters Description

  • reason (str): The reason why the test is expected to fail.

Example
