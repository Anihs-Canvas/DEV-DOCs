📊 Constants

Pytest constants provide version information and special parameters that can be used throughout your test suite. These constants help you write version-aware tests and control test naming behavior.

pytest.__version__

Full Description

The pytest.__version__ constant returns the current installed version of pytest as a string. This is useful for checking pytest version compatibility in your test suite or when debugging test failures that might be version-specific. The version string follows semantic versioning (MAJOR.MINOR.PATCH) format.

Syntax

import pytest
version = pytest.__version__

Parameters Description

This constant does not accept any parameters. It is a read-only string attribute that contains the version information.

Attributes

Type: str

Format: MAJOR.MINOR.PATCH (e.g., "7.0.0")

Access: Read-only attribute

Module: pytest

Methods

This constant has no methods. It is a simple string constant. However, you can use standard Python string methods on it such as:

  • .split('.') - Split version into components
  • .startswith() - Check version prefix
  • comparison operators - Compare versions (use packaging.version for proper comparison)

Return Value

Returns: A string representing the current pytest version in the format "MAJOR.MINOR.PATCH"

Example return values: "7.0.0", "7.4.3", "8.0.0rc1"

pytest.HIDDEN_PARAM

Full Description

The pytest.HIDDEN_PARAM constant (added in pytest 8.4) is a special marker that can be passed to the ids parameter of pytest.mark.parametrize() or to the id parameter of pytest.param(). When used, it hides a parameter set from the test name, which is useful for reducing test name verbosity while maintaining unique test identifiers. This can only be used at most once per parametrize call since test names must remain unique.

Syntax

import pytest

@pytest.mark.parametrize(
    "param1,param2",
    [
        (value1, value2),
        pytest.param(value3, value4, id=pytest.HIDDEN_PARAM)
    ]
)

Parameters Description

This constant does not accept any parameters. It serves as a sentinel value to indicate that a parameter set should be hidden from test names.

Attributes

Type: Special constant marker

Usage: Can be used in ids parameter of parametrize or id parameter of pytest.param()

Limitation: Can only be used at most once per parametrize call

Module: pytest

Version: Added in pytest 8.4

Methods

This constant has no methods. It is a marker constant used for controlling test naming behavior.

Return Value

Returns: Not applicable - this is a constant marker, not a function. When used, it causes the associated parameter set to be hidden from the generated test name.

pytest.version_tuple

Full Description

The pytest.version_tuple constant (added in pytest 7.0) returns the current pytest version as a tuple of integers and strings. This is more convenient for version comparisons than parsing the string version. The tuple format is (MAJOR, MINOR, PATCH) where each component is an integer. For pre-releases, the last component will be a string like "0rc1".

Syntax

import pytest
version_tuple = pytest.version_tuple

Parameters Description

This constant does not accept any parameters. It is a read-only tuple attribute that contains the version components.

Attributes

Type: tuple

Format: (MAJOR: int, MINOR: int, PATCH: int|str)

Example: (7, 4, 3) or (8, 0, '0rc1')

Access: Read-only tuple

Module: pytest

Version: Added in pytest 7.0

Methods

This constant has no specific methods, but as a tuple you can use standard tuple operations:

  • [index] - Access individual components (e.g., version_tuple[0] for major)
  • comparison operators - Direct tuple comparison (e.g., version_tuple >= (7, 0, 0))
  • unpacking - Unpack into variables (e.g., major, minor, patch = version_tuple)

Return Value

Returns: A tuple containing version components as integers (or string for pre-release suffix)

Example values: (7, 4, 3), (8, 0, 0), (8, 0, '0rc1')
